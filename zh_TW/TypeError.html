類別public dynamic class TypeError
繼承TypeError Error Object

語言版本: ActionScript 3.0
執行階段版本: Flash Player 9, AIR 1.0, Flash Lite 4

當運算元的實際類型不同於預期的類型時，便會擲回 TypeError 例外。

此外，這個例外也會在以下情況中擲回：

  • 無法將函數或方法的實際參數強制轉型為正式的參數類型。
  • 將某個值指派給變數，而且無法將該值強制轉型為變數的類型。
  • is 或 instanceof 運算子的右側不是有效的類型。
  • 以不合法的方式使用 super 關鍵字。
  • 屬性查閱造成一個以上的繫結，因而模糊不清。
  • 對不相容的物件叫用方法。 例如，如果將某個 RegExp 類別方法「移植」到一般物件，然後再叫用此方法，便會擲回 TypeError 例外。

範例 如何使用本範例
TypeErrorExample.as

下列範例會示範如何產生 TypeError 例外以及在 try..catch 陳述式中處理。 
package {
    import flash.display.DisplayObject;
    import flash.display.Sprite;

    public class TypeErrorExample extends Sprite {
        public function TypeErrorExample() {
            try {
                var child:Object = new Object();
                addChild(DisplayObject(child));
            }
            catch(e:TypeError) {
                trace(e);
            }
        }
    }
}
