Compilerwaarschuwingen identificeren code die geldig is en correct wordt gecompileerd, maar mogelijk niet functioneert zoals de maker heeft bedoeld. Wanneer u het detecteren van deze mogelijke problemen wilt inschakelen, compileert u ActionScript-projecten in de waarschuwingsmodus.

Sommige van deze waarschuwingen (bijvoorbeeld 'Typedeclaratie ontbreekt.') hebben te maken met keuzes in codestijl waaraan u zich wel of niet kunt houden. Andere waarschuwingen (bijvoorbeeld 'Onmogelijke toewijzing naar null.') geven instructies aan die geldig zijn, maar waarschijnlijk niet doen wat de gebruiker verwacht. Een derde categorie waarschuwingen heeft betrekking op problemen die kunnen optreden bij het overbrengen van code van ActionScript 2.0 naar ActionScript 3.0.

 3596Dubbele variabeledefinitie. 
 3597Dubbele variabeledefinitie. De compiler heeft een dubbele definitie van een variabele aangetroffen. Dit kan tot onverwachte resultaten leiden. ActionScript ondersteunt geen bereik op het blokniveau voor variabelen. Alle variabelen die binnen de hoofdtekst van een functie zijn gedefinieerd, bestaan binnen hetzelfde bereik, zelfs als ze zijn gedefinieerd binnen een if-instructie, while-instructie, for-instructie enzovoort. De volgende code, bijvoorbeeld, declareert de variabele x tweemaal: 

function test() {
	var x:Number = 10;
	if (true) {
	    for (var x=0; x < 5; x++)  // warning here, this is the second defintion of x
	    trace(x);
	}
	trace(x); // 5, not 10.  The last value set by the for loop above is the current value of x
}
