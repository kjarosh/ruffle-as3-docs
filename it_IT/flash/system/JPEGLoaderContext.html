Pacchettoflash.system
Classepublic class JPEGLoaderContext
EreditarietàJPEGLoaderContext LoaderContext Object

Versione linguaggio: ActionScript 3.0
Versioni runtime: Flash Player 10, AIR 1.5, Flash Lite 4

La classe JPEGLoaderContext include una proprietà che consente di abilitare un filtro di deblock durante il caricamento di un'immagine JPEG. Il filtro di deblock migliora la qualità di un'immagine con impostazioni di compressione più elevate attenuando i pixel adiacenti. Per applicare il deblock quando caricate un'immagine JPEG, create un oggetto JPEGLoaderContext e impostatene la proprietà deblockingFilter. Quindi, utilizzate il nome dell'oggetto JPEGLoaderContext come valore del parametro context del metodo load() dell'oggetto Loader utilizzato per caricare l'immagine.

La classe JPEGLoaderContext estende la classe LoaderContext. Impostate la proprietà checkPolicyFile su true se vi serve l'accesso a livello di codice ai pixel dell'immagine caricata (ad esempio, se utilizzate il metodo BitmapData.draw()). Non è necessario impostare la proprietà checkPolicyFile per il contenuto AIR in esecuzione nella sandbox dell'applicazione.

Descrizione delle proprietà

deblockingFilter

proprietà
public var deblockingFilter:Number = 0.0

Versione linguaggio: ActionScript 3.0
Versioni runtime: Flash Player 10, AIR 1.5, Flash Lite 4

Specifica la potenza del filtro di deblock. Il valore 1.0 applica un filtro di deblock a potenza piena, mentre il valore 0.0 disabilita il filtro di deblock.

Descrizione della funzione di costruzione

JPEGLoaderContext

()Funzione di costruzione
public function JPEGLoaderContext(deblockingFilter:Number = 0.0, checkPolicyFile:Boolean = false, applicationDomain:ApplicationDomain = null, securityDomain:SecurityDomain = null)

Versione linguaggio: ActionScript 3.0
Versioni runtime: Flash Player 10, AIR 1.5, Flash Lite 4

Crea un nuovo oggetto JPEGLoaderContext con le impostazioni specificate.

Parametri
deblockingFilter:Number (default = 0.0) — Specifica la potenza del filtro di deblock. Il valore 1.0 applica un filtro di deblock a potenza piena, mentre il valore 0.0 disabilita il filtro di deblock.
 
checkPolicyFile:Boolean (default = false) — Specifica se Flash Player deve verificare l'esistenza di un file di criteri degli URL prima di caricare l'oggetto. Non si applica al contenuto AIR nella sandbox di sicurezza dell'applicazione.
 
applicationDomain:ApplicationDomain (default = null) — Specifica l'oggetto ApplicationDomain da utilizzare per un oggetto Loader.
 
securityDomain:SecurityDomain (default = null) — Specifica l'oggetto SecurityDomain da utilizzare per un oggetto Loader.
