Descrizione delle proprietà

authenticationMethod

proprietà
authenticationMethod:String  [sola lettura]

Versione linguaggio: ActionScript 3.0
Versioni runtime: AIR 2, Flash Player 10.1

Il tipo di autenticazione richiesto per ottenere un voucher per il contenuto associato.

I tipi di autenticazione supportati sono i seguenti:

  • AuthenticationMethod.ANONYMOUS - chiunque può ottenere un voucher.
  • AuthenticationMethod.USERNAME_AND_PASSWORD: l'utente deve fornire un nome utente e una password validi per un account autorizzato a visualizzare il contenuto associato.

La classe AuthenticationMethod fornisce le costanti di stringa da utilizzare con la proprietà authenticationMethod.

displayName

proprietà 
displayName:String  [sola lettura]

Versione linguaggio: ActionScript 3.0
Versioni runtime: AIR 2, Flash Player 10.1

Una stringa in formato comprensibile che potete utilizzare per fare riferimento all'oggetto VoucherAccessInfo specificato nell'interfaccia utente.

Se il file di metadati per un contenuto multimediale dispone di più voucher, ognuno con il proprio oggetto VoucherAccessInfo, l'utente potrebbe dover decidere per quale voucher autenticarsi. Potreste disporre, ad esempio di un voucher a livello di sottoscrizione con privilegi elevati per la visualizzazione del contenuto e di un voucher a livello di base con privilegi inferiori. Per distinguere questi due voucher, utilizzate la stringa descrittiva nella proprietà displayName. La stringa viene impostata dal packager di contenuti multimediali (lo strumento che crea i pacchetti di contenuti multimediali e li crittografa per prepararli alla distribuzione tramite un server di gestione dei diritti multimediali, ad esempio Flash Access).

Per le applicazioni localizzate, potete utilizzare questa proprietà come identificatore. L'applicazione può rilevare le impostazioni internazionali e localizzare la stringa di conseguenza.

domain

proprietà 
domain:String  [sola lettura]

Versione linguaggio: ActionScript 3.0
Versioni runtime: AIR 2, Flash Player 10.1

Il dominio del contenuto del server di gestione dei diritti multimediali presso il quale l'utente deve autenticarsi per ottenere il voucher per il contenuto associato.

Se l'autenticazione viene effettuata sul dominio predefinito o se non è richiesta alcuna autenticazione, il valore di domainName è null.

Nota: il dominio restituito da questa proprietà non ha nulla a che fare con i nomi di dominio di rete o Internet. In questo caso, con il termine dominio si indica un gruppo di account contenuti o utenti. Ad esempio, un singolo server può supportare diversi domini, ognuno dotato di un proprio set di canali di contenuti e iscritti.

policyID

proprietà 
policyID:String  [sola lettura]

Versione linguaggio: ActionScript 3.0
Versioni runtime: AIR 3.8, Flash Player 11.8

Un identificatore univoco per il criterio rappresentato da VoucherAccessInfo.

Se il file di metadati per un contenuto multimediale dispone di più voucher, ognuno con il proprio oggetto VoucherAccessInfo, l'utente potrebbe dover decidere per quale voucher autenticarsi. Potreste disporre, ad esempio di un voucher a livello di sottoscrizione con privilegi elevati per la visualizzazione del contenuto e di un voucher a livello di base con privilegi inferiori. policyID dell’oggetto DRMVoucher acquisito corrisponderà a policyID di VoucherAccessInfo in cui è stato emesso il voucher.
