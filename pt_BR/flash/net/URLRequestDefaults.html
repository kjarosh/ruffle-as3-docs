Classepublic class URLRequestDefaults
HerançaURLRequestDefaults Object

Versões de runtime: AIR 1.0, Flash Lite 4

A classe URLRequestDefaults inclui propriedades estáticas que podem ser configuradas para definir os valores padrão das propriedades da classe URLRequest. Ela também inclui um método estático, URLRequestDefaults.setLoginCredentialsForHost(), que permite que você defina credenciais de autenticação padrão para a solicitação. A classe URLRequest define as informações a serem usadas numa solicitação HTTP.

Quaisquer propriedades definidas em um objeto URLRequest sobrescrevem as propriedades estáticas definidas para a classe URLRequestDefaults.

As configurações de URLRequestDefault se aplicam somente ao conteúdo no domínio do aplicativo chamador, com uma exceção: as configurações feitas pela chamada URLRequestDefaults.setLoginCredentialsForHost() se aplicam a todos os domínios de aplicativo no aplicativo em execução atualmente.

Apenas o conteúdo do Adobe® AIR™ em execução na caixa de proteção do aplicativo poderá usar a classe URLRequestDefaults. O outro conteúdo resultará em um SecurityError sendo lançado ao acessar os membros e propriedades dessa classe.

idleTimeout

propriedade 
idleTimeout:Number

Versão da linguagem: ActionScript 3.0
Versões de runtime: AIR 2

A configuração padrão da propriedade idleTimeout dos objetos URLRequest e objetos HTMLLoader.

O tempo limite ocioso é o período de tempo (em milissegundos) que o cliente espera por uma resposta do servidor, depois que a conexão é estabelecida, antes de abandonar o pedido.

Isto define o tempo limite ocioso padrão usado pelo objeto URLRequest ou HTMLLoader. Configurar a propriedade idleTimeout em um objeto URLRequest ou um objeto de HTMLLoader sobrescreve esta configuração padrão.

Quando esta propriedade é definida como 0 (o padrão), o tempo de execução usa o valor padrão de tempo limite ocioso definido pelo sistema operacional. O valor padrão de tempo limite ocioso varia entre sistemas operacionais (como Mac OS, Linux ou Windows) e entre versões do sistema operacional.

Esta configuração não se aplica aos objetos URLRequest usados no upload de arquivos ou a solicitações RTMP.

O valor padrão é 0.

userAgent

propriedade 
userAgent:String

Versões de runtime: AIR 1.0, Flash Lite 4

As configurações padrão para a propriedade userAgent dos objetos URLRequest. A configuração da propriedade userAgent em um objeto URLRequest sobrescreve esta configuração padrão.

Esta também é a string padrão "user agent" para todos os objetos HTMLLoader (usados ao chamar o método load()do objeto HTMLLoader). A configuração da propriedade userAgent do objeto HTMLLoader sobrescreve a configuração URLRequestDefaults.userAgent.

Esse valor padrão varia em função do sistema operacional do tempo de execução (tal como Mac OS, Linux ou Windows), do idioma do tempo de execução e da versão, como nos dois exemplos a seguir:

  • "Mozilla/5.0 (Macintosh; U; PPC Mac OS X; en) AppleWebKit/526.9+ (KHTML, como Gecko) AdobeAIR/1.5"
  • "Mozilla/5.0 (Windows; U; en) AppleWebKit/526.9+ (KHTML, como Gecko) AdobeAIR/1.5"
  • "Mozilla/5.0 (X11; U; Linux i686; en-US) AppleWebKit/526.9+ (KHTML, como Gecko) AdobeAIR/1.5"

Detalhes do método

setLoginCredentialsForHost

()método
public static function setLoginCredentialsForHost(hostname:String, user:String, password:String):*

Versões de runtime: AIR 1.0, Flash Lite 4

Define as credenciais de usuário padrão e a senha para o host selecionado. Essas configurações se aplicam a objetos URLRequest em todos os domínios de aplicativo, não apenas no domínio do objeto que chama o método (enquanto as propriedades estáticas da classe URLRequest só se aplicam ao domínio de aplicativo do chamador). Isso permite que o conteúdo em todo o aplicativo (independentemente do domínio de aplicativo do conteúdo) efetue login quando a outra parte do aplicativo o fizer.

Observação sobre aplicativos executados no Mac OS: No Mac OS, quando você chama esse método, o aplicativo usa essas credenciais para o host especificado até ser fechado, mesmo que depois você chame URLRequestDefaults.setLoginCredentialsForHost() para o mesmo host. Contudo, se um servidor rejeitar as credenciais especificadas por este método, então uma chamada subsequente ao método URLRequestDefaults.setLoginCredentialsForHost() (para o mesmo host) será reconhecida.

Observação:Este método não se aplica aos objetos URLRequests usados no upload de arquivos ou solicitações RTMP.

Parâmetros

hostname:String — O nome do host no qual o usuário e senha são aplicados. Pode ser um domínio, como "www.exemplo.com.br" ou um domínio e número de porta, tal como "www.exemplo.com.br:80". Observe que "exemplo.com.br", "exemplo.com.br" e "vendas.com.br" são todos considerados hosts únicos.
 
user:String — O nome de usuário padrão na solicitação de autenticação para o host especificado.
 
password:String — A senha padrão usada na solicitação de autenticação para o host especificado.
