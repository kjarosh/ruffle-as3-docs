Klasapublic class StageVideoEvent
DziedziczenieStageVideoEvent Event Object

Wersja języka: ActionScript 3.0
Wersje środowiska wykonawczego: AIR 2.5, Flash Player 10.2

Obiekt StageVideo wywołuje obiekt StageVideoEvent po tym, jak została wywołana zarówno metoda attachNetStream() obiektu StageVideo, jak i metoda play() dołączonego obiektu NetStream. Oprócz tego, zależnie od platformy, każda zmiana stanu odtwarzania może skutkować wywołaniem zdarzenia. Jednym z typów zdarzenia StageVideoEvent jest StageVideoEvent.RENDER_STATE.

()Konstruktor
public function StageVideoEvent(type:String, bubbles:Boolean = false, cancelable:Boolean = false, status:String = null, colorSpace:String = null)

Wersja języka: ActionScript 3.0
Wersje środowiska wykonawczego: AIR 2.5, Flash Player 10.2

Tworzy obiekt Event, który zawiera informacje o zdarzeniach StageVideo. Obiekty Event są przekazywane jako parametry do detektorów zdarzeń.

Parametry
type:String — Typ zdarzenia. Detektory zdarzeń mogą uzyskiwać dostęp do tej informacji za pośrednictwem dziedziczonej właściwości type. Jednym z typów zdarzenia StageVideoEvent jest StageVideoEvent.RENDER_STATE.
 
bubbles:Boolean (default = false) — Określa, czy obiekt Event uczestniczy w etapie propagacji przepływu zdarzeń. Detektory zdarzeń mogą uzyskiwać dostęp do tej informacji za pośrednictwem dziedziczonej właściwości bubbles.
 
cancelable:Boolean (default = false) — Określa, czy obiekt Event może zostać anulowany. Detektory zdarzeń mogą uzyskiwać dostęp do tej informacji za pośrednictwem dziedziczonej właściwości cancelable.
 
status:String (default = null) — Wskazuje stan obiektu docelowego StageVideo.
 
colorSpace:String (default = null) — Przestrzeń kolorów używana przez wyświetlane wideo.

RENDER_STATUS_UNAVAILABLE

Stała 
public static const RENDER_STATUS_UNAVAILABLE:String = "unavailable"
Przestarzałe od Flash Player 10.2, AIR 3: Użyj flash.media.VideoStatus.UNAVAILABLE

Wersja języka: ActionScript 3.0
Wersje środowiska wykonawczego: AIR 2.5

Wskazuje, że nie jest możliwe wyświetlenie wideo przy użyciu obiektu StageVideo.

Jest to jedna z możliwych wartości właściwości status obiektu StageVideoEvent.

Załóżmy na przykład, że platforma nie obsługuje dekodowania i wyświetlania wideo (kodeka dźwiękowego lub wideo) ani przy użyciu oprogramowania, ani sprzętu. W takim przypadku obiekt StageVideoEvent ma podaną tę wartość właściwości „status”.

Ta wartość właściwości „status” jest również używana, gdy nie są dostępne żadne dekodery sprzętowe. Taka sytuacja może wystąpić w środowisku AIR dla urządzeń telewizyjnych. W celu uzyskania zgodności z wcześniejszymi wersjami środowisko AIR dla urządzeń telewizyjnych umożliwia dekodowanie i wyświetlanie sprzętowe przy użyciu obiektu Video. W przypadku zastosowania obiektu Video jest używany podstawowy dekoder sprzętowy, dlatego jeden dodatkowy obiekt StageVideo staje się niedostępny do użytku. Zalecane jest, aby do sprzętowego dekodowania i wyświetlania wideo używać obiektu StageVideo.
