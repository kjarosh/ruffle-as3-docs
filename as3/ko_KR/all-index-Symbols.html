@ (attribute identifier) — 연산자
XML 또는 XMLList 객체의 특성을 식별합니다.
+ (addition) — 연산자
숫자 표현식을 추가합니다.
+= (addition assignment) — 연산자
expression1에 expression1 + expression2의 값을 대입합니다.
[] (array access) — 연산자
a0 등과 같은 지정된 요소를 사용하여 새 배열 또는 다차원 배열을 초기화하거나 배열의 요소에 액세스합니다.
= (assignment) — 연산자
expression2의 값(오른쪽 피연산자)을 expression1의 변수, 배열 요소 또는 속성에 대입합니다.
& (bitwise AND) — 연산자
expression1 및 expression2를 32비트 부호 없는 정수로 변환하고 정수 매개 변수의 각 비트에 대해 부울 AND 연산을 수행합니다.
&= (bitwise AND assignment) — 연산자
expression1에 expression1 & expression2의 값을 대입합니다.
<< (bitwise left shift) — 연산자
expression1 및 shiftCount를 32비트 정수로 변환하고, expression1의 모든 비트를 shiftCount 변환 결과 정수값으로 지정된 자릿수만큼 왼쪽으로 이동합니다.
<<= (bitwise left shift and assignment) — 연산자
bitwise left shift(<<=) 연산을 수행하고 그 결과를 expression1에 저장합니다.
~ (bitwise NOT) — 연산자
expression을 32비트 부호 있는 정수로 변환한 다음 비트 1의 보수를 적용합니다.
| (bitwise OR) — 연산자
expression1 및 expression2를 32비트 부호 없는 정수로 변환하고, expression1 또는 expression2의 해당 비트가 1인 각 비트 위치에 1을 배치합니다.
|= (bitwise OR assignment) — 연산자
expression1에 expression1 | expression2의 값을 대입합니다.
>> (bitwise right shift) — 연산자
expression 및 shiftCount를 32비트 정수로 변환하고, expression의 모든 비트를 shiftCount의 변환 결과로 나온 정수로 지정된 자릿수만큼 오른쪽으로 이동합니다.
>>= (bitwise right shift and assignment) — 연산자
bitwise right shift 연산을 수행하고 그 결과를 expression에 저장합니다.
>>> (bitwise unsigned right shift) — 연산자
왼쪽 비트에는 항상 0이 채워지므로 원래 표현식의 부호가 유지되지 않는다는 점을 제외하면 bitwise right shift(>>) 연산자와 동일합니다.
>>>= (bitwise unsigned right shift and assignment) — 연산자
부호 없는 bitwise right shift 연산을 수행하고 그 결과를 expression에 저장합니다.
^ (bitwise XOR) — 연산자
expression1 및 expression2를 32비트 부호 없는 정수로 변환하고, expression1 또는 expression2 중 하나의 해당 비트가 1인 각 비트 위치에 1을 배치합니다.
^= (bitwise XOR assignment) — 연산자
expression1에 expression1 ^ expression2의 값을 대입합니다.
/*..*/ (block comment delimiter) — 연산자
한 줄 이상의 스크립트 주석을 구분합니다.
{ } (braces (XML)) — 연산자
XML 또는 XMLList 이니셜라이저에 사용된 표현식을 평가합니다.
[ ] (brackets (XML)) — 연산자
XML 또는 XMLList 객체의 속성 또는 특성에 액세스합니다.
, (comma) — 연산자
expression1, expression2 등의 순서대로 평가합니다.
+ (concatenation) — 연산자
문자열을 연결 또는 결합합니다.
+= (concatenation assignment) — 연산자
expression1에 expression1 + expression2의 값을 대입합니다.
+= (concatenation assignment (XMLList)) — 연산자
expression1에 XMLList 객체인 expression1 + expression2의 값을 대입합니다.
+ (concatenation (XMLList)) — 연산자
XML 또는 XMLList 값을 XMLList 객체로 연결(결합)합니다.
?: (conditional) — 연산자
expression1을 평가하여 expression1의 값이 true이면 결과는 expression2의 값이고, 그렇지 않으면 결과는 expression3의 값입니다.
-- (decrement) — 연산자
피연산자에서 1을 뺍니다.
.. (descendant accessor) — 연산자
XML 또는 XMLList 객체의 자손 요소로 이동하거나 @ 연산자와 결합된 경우 자손의 일치하는 특성을 찾습니다.
/ (division) — 연산자
expression1을 expression2로 나눕니다.
/= (division assignment) — 연산자
expression1에 expression1 / expression2의 값을 대입합니다.
. (dot) — 연산자
클래스 변수 및 메서드에 액세스하고, 객체 속성을 가져오고 설정하며, 가져온 패키지 또는 클래스를 구분합니다.
. (dot (XML)) — 연산자
XML 또는 XMLList 객체의 자식 요소로 이동하거나 @ 연산자와 결합된 경우 XML 또는 XMLList 객체의 특성을 반환합니다.
== (equality) — 연산자
두 표현식이 동일한지 여부를 테스트합니다.
> (greater than) — 연산자
두 표현식을 비교하여 expression1이 expression2보다 큰지 여부를 확인합니다. 큰 경우 결과는 true입니다.
>= (greater than or equal to) — 연산자
두 표현식을 비교하여 expression1이 expression2보다 크거나 같은지(true) 또는 expression1이 expression2보다 작은지(false) 확인합니다.
++ (increment) — 연산자
표현식에 1을 더합니다.
!= (inequality) — 연산자
equality(==) 연산자의 정반대 상황을 테스트합니다.
< (less than) — 연산자
두 표현식을 비교하여 expression1이 expression2보다 작은지 여부를 확인합니다. 작은 경우 결과는 true입니다.
<= (less than or equal to) — 연산자
두 표현식을 비교하여 expression1이 expression2보다 작거나 같은지를 확인합니다. 작거나 같은 경우 결과는 true입니다.
// (line comment delimiter) — 연산자
스크립트 주석의 시작을 나타냅니다.
&& (logical AND) — 연산자
false이거나 false로 변환될 수 있으면 expression1을 반환하고, 그렇지 않으면 expression2를 반환합니다.
&&= (logical AND assignment) — 연산자
expression1에 expression1 && expression2의 값을 대입합니다.
! (logical NOT) — 연산자
변수 또는 표현식의 부울 값을 반대로 합니다.
|| (logical OR) — 연산자
true이거나 true로 변환될 수 있으면 expression1을 반환하고, 그렇지 않으면 expression2를 반환합니다.
||= (logical OR assignment) — 연산자
expression1에 expression1 || expression2의 값을 대입합니다.
% (modulo) — 연산자
expression1을 expression2로 나눈 나머지를 계산합니다.
%= (modulo assignment) — 연산자
expression1에 expression1 % expression2의 값을 대입합니다.
* (multiplication) — 연산자
두 개의 숫자 표현식을 곱합니다.
*= (multiplication assignment) — 연산자
expression1에 expression1 * expression2의 값을 대입합니다.
:: (name qualifier) — 연산자
속성, 메서드, XML 속성 또는 XML 특성의 네임스페이스를 식별합니다.
{} (object initializer) — 연산자
지정된 name 및 value 속성 쌍으로 새 객체를 만들거나 초기화합니다.
() (parentheses) — 연산자
하나 이상의 매개 변수에 대해 그룹화 연산을 수행하고, 표현식을 연속적으로 평가하거나, 하나 이상의 매개 변수를 묶은 다음 괄호 앞의 함수에 인수로 전달합니다.
( ) (parentheses (XML)) — 연산자
E4X XML 구문의 표현식을 평가합니다.
/ (RegExp delimiter) — 연산자
문자의 앞/뒤에 사용하면 해당 문자에 리터럴 값이 있으므로 이 문자를 변수, 문자열 또는 다른 ActionScript 요소가 아니라 일반 표현식(RegExp)으로 간주해야 함을 나타냅니다.
=== (strict equality) — 연산자
두 표현식이 동일한지 테스트합니다. 그러나 자동 데이터 변환은 수행하지 않습니다.
!== (strict inequality) — 연산자
strict equality(===) 연산자의 정반대 상황을 테스트합니다.
" (string delimiter) — 연산자
문자의 앞/뒤에 사용하면 해당 문자에 리터럴 값이 있으므로 이 문자를 변수, 숫자 값 또는 다른 ActionScript 요소가 아니라 문자열로 간주해야 함을 나타냅니다.
- (subtraction) — 연산자
부정하거나 빼는 데 사용됩니다.
-= (subtraction assignment) — 연산자
expression1에 expression1 - expression2의 값을 대입합니다.
symbolsButtonBar — 스킨 파트, 클래스 com.adobe.solutions.exm.authoring.components.toolbars.EBToolBar
A Skin part which defines UI component to display Symbols ButtonBar. A Skin part which defines UI component to display Symbols ButtonBar.
: (type) — 연산자
데이터 유형 대입에 사용되는 이 연산자는 변수 유형, 함수 반환 유형 또는 함수 매개 변수 유형을 지정합니다.
< > (XML literal tag delimiter) — 연산자
XML 리터럴의 XML 태그를 정의합니다.

ActionScript 3.0 참조 설명서의 내용이 영어로 나타납니다.

ActionScript 3.0 참조 설명서 중 일부는 전체 언어로 번역되지 않았습니다. 언어 요소가 번역되지 않은 경우 영어로 나타납니다. 예를 들어 ga.controls.HelpBox 클래스는 어떤 언어로도 번역되지 않았습니다. 그러므로 한국어 버전의 참조 설명서에서 ga.controls.HelpBox 클래스는 영어로 나타납니다.
