상속FontDescription Object

언어 버전: ActionScript 3.0
런타임 버전: Flash Player 10, AIR 1.5, Flash Lite 4

FontDescription 클래스는 글꼴을 설명하는 데 필요한 정보를 나타냅니다.

FontDescription 객체가 ElementFormat에 적용되고 ElementFormat이 다시 TextBlock의 ContentElement에 적용됩니다. FontDescription을 ElementFormat에 적용하면 locked 속성이 true로 설정됩니다. 잠긴 FontDescription 객체의 속성은 변경할 수 없습니다. 대신 clone() 메서드를 사용하여 객체의 잠기지 않은 복사본을 만들면 수정하거나 ElementFormat에 할당할 수 있습니다.

참고: FTE(Flash Text Engine)는 Type 1 글꼴이나 Type 3, ATC, sfnt-wrapped CID 또는 Naked CID와 같은 비트맵 글꼴을 지원하지 않습니다.

cffHinting

cffHinting:String

언어 버전: ActionScript 3.0
런타임 버전: Flash Player 10, AIR 1.5, Flash Lite 4

이 텍스트에 사용된 CFF 힌팅 유형입니다. 이 속성에는 CFFHinting 상수를 사용합니다. 이 속성은 텍스트의 RenderingMode 속성을 RenderingMode.CFF로 설정한 경우에만 적용됩니다.

Flash 런타임에서 굵은 가로 스템을 하위 픽셀 격자에 강제로 맞출 것인지 여부를 결정하는 데 사용되는 CFF(Compact Font Format) 힌팅의 유형입니다.

포함된 글꼴에만 적용됩니다.

기본값은 CFFHinting.HORIZONTAL_STEM입니다.

CFFHinting 속성의 경우 CFFHinting 클래스에서 다음 상수를 사용할 수 있습니다.

문자열 값설명
CFFHinting.NONECFF 힌팅을 지정하지 않습니다. 글리프의 가로 스템을 하위 픽셀 격자에 강제로 맞추지 않습니다. 이 설정은 애니메이션이나 큰 글꼴 크기에 적합합니다.
CFFHinting.HORIZONTAL_STEMCFF 힌팅을 지정합니다. 굵은 가로 스템이 화면의 하위 픽셀 격자에 맞춰집니다. 이 설정을 사용하려면 RenderingMode 속성을 RenderingMode.CFF로 설정해야 합니다.

fontName

속성 
fontName:String

언어 버전: ActionScript 3.0
런타임 버전: Flash Player 10, AIR 1.5

사용할 글꼴의 이름 또는 쉼표로 구분한 글꼴 이름 목록입니다. 런타임은 목록 중 첫 번째로 사용 가능한 글꼴 요소를 렌더링합니다. 예를 들어 목록이 "Arial, Helvetica, _sans"인 경우 플레이어는 Arial을 검색하고 Arial이 없을 경우 Helvetica를 검색하며 둘 다 없을 경우 _sans를 검색합니다.

Flash 런타임은 _sans(sans serif 글꼴의 경우), _serif(serif 글꼴의 경우), _typewriter(monospace 글꼴의 경우) 등 세 가지 일반 장치 글꼴 이름을 지원합니다. 이러한 이름은 플랫폼에 따라 특정 장치의 글꼴로 매핑됩니다.

기본값은 "_serif"입니다.

Flash 런타임은 선택한 글꼴에 포함되지 않은 글리프에 대해 대체 글꼴을 제공합니다. 포함된 글꼴을 사용하는지 아니면 장치 글꼴을 사용하는지 여부와 상관없이 글꼴에 해당 글리프가 없으면 런타임은 해당 글리프가 들어 있을 가능성이 높은 다른 장치 글꼴을 사용하여 렌더링을 시도합니다.

예제  ( 예제 사용 방법 )

다음은 FontDescription 객체를 만들고 장치 글꼴을 해당 객체에 할당하고 여러 가지 글꼴 속성을 설정하고 새 객체를 ElementFormat 객체에 할당하여 잠그는 예제입니다. 새 TextElement를 만들고 이때 ElementFormat을 사용합니다. TextElement는 TextBlock의 내용으로 사용됩니다. TextBlock에서 만들어진 텍스트 줄입니다.

FontDescription 객체를 수정하려면 먼저 locked 속성을 선택합니다. true이면 clone() 메서드를 사용하여 FontDescription의 잠기지 않은 복사본을 만들고 속성을 변경한 다음 새 ElementFormat 객체에 할당합니다. 그런 다음 새 ElementFormat을 TextBlock에 다시 연결합니다. 줄을 다시 줄 바꿈할 때 새 줄에 새 글꼴 설정이 적용됩니다.


package {

    import flash.display.Sprite;
    import flash.text.engine.*;

    
    public class FontDescription_cloneExample extends Sprite {
        private var fd:FontDescription;
        private var fd2:FontDescription;
        
        public function FontDescription_cloneExample():void {
            fd = new FontDescription();
            fd.fontLookup = flash.text.engine.FontLookup.DEVICE;
            fd.fontName = "Palatino";
            fd.fontWeight = flash.text.engine.FontWeight.BOLD;
            fd.fontPosture = flash.text.engine.FontPosture.ITALIC;

            var ef1:ElementFormat = new ElementFormat(fd);
            
            var str:String = "This is flash text 0123456789";
            var tb:TextBlock = new TextBlock();
            var te1:TextElement = new TextElement(str, ef1);
            tb.content = te1;
            var line1:TextLine = tb.createTextLine(null, 600);
            addChild(line1);
            
            fd2 = (fd.locked) ? fd.clone() : fd;    
            fd2.fontWeight = flash.text.engine.FontWeight.NORMAL;
            var ef2:ElementFormat = new ElementFormat(fd2);

            tb.content.elementFormat=ef2;
            var line2:TextLine = tb.createTextLine(null, 600);
            addChild(line2);
            
        }
    }    
}

지정된 fontName, fontWeight 및 fontPosture와 사용 가능한 장치 글꼴이 있는 경우 true를 반환합니다.

flash.text.engine 클래스는 OpenType 및 TrueType 장치 글꼴만 사용할 수 있습니다. 이전의 글꼴 기술을 기반으로 하는 글꼴이 사용되는 경우 런타임은 텍스트를 렌더링하는 데 글리프별로 알려진 적합한 장치 글꼴을 대신 사용합니다.

매개 변수

fontName:String — 확인할 장치 글꼴의 이름입니다.
 
fontWeight:String — 글꼴 굵기를 지정합니다. FontWeight를 사용합니다.
 
fontPosture:String — 글꼴 모양을 지정합니다. FontPosture를 사용합니다.

반환값
Boolean — 호환되는 장치 글꼴을 사용할 수 있으면 true이고, 그렇지 않으면 false입니다.

오류
ArgumentError — 지정된 fontWeight가 FontWeight의 멤버가 아닙니다.
 
ArgumentError — 지정된 fontPosture가 FontPosture의 멤버가 아닙니다.

언어 버전: ActionScript 3.0
런타임 버전: Flash Player 10, AIR 1.5, Flash Lite 4

지정된 fontName, fontWeight 및 fontPosture로 포함된 글꼴을 사용할 수 있고 Font.fontType이 flash.text.FontType.EMBEDDED_CFF이면 true를 반환합니다. Flash Player 10 이상에서는 두 가지 유형의 포함된 글꼴이 응용 프로그램 내용에 나타날 수 있습니다. 일반적인 포함된 글꼴은 TextField에만 사용됩니다. CFF 포함 글꼴은 flash.text.engine 클래스에만 사용됩니다. 이러한 두 가지 유형은 fontType 속성(Font 클래스)에 따라 구분되며, 이 클래스는 enumerateFonts() 함수로 반환됩니다.

flash.text.engine 클래스에서는 유형이 EMBEDDED인 글꼴을 사용할 수 없습니다. fontLookup이 FontLookup.EMBEDDED_CFF로 설정되어 있고 런타임에 지정된 이름, 굵기 및 모양으로 사용 가능한 글꼴이 EMBEDDED 유형 글꼴뿐인 경우 런타임은 글리프별로 장치 글꼴을 대신 사용하여 텍스트를 렌더링합니다. 이 동작은 지정된 이름과 스타일로 포함된 글꼴을 사용할 수 없는 경우와 같습니다.

이름, 굵기 및 모양이 동일한 EMBEDDED와 EMBEDDED_CFF 글꼴을 모두 사용할 수 있으면 EMBEDDED_CFF 글꼴이 선택되고 텍스트가 EMBEDDED_CFF 글꼴로 렌더링됩니다.

fontName:String — 확인할 포함된 글꼴의 이름입니다.

예제 예제 사용 방법
FontDescriptionExample.as

다음은 FontDescription 객체를 만들고 장치 글꼴을 해당 객체에 할당하고 여러 가지 글꼴 속성을 설정하고 새 객체를 ElementFormat 객체에 할당하는 예제입니다. ElementFormat 내에서 글꼴 서식을 추가로 설정합니다. 새 TextElement를 만들고 이때 ElementFormat을 사용합니다. TextElement는 TextBlock의 내용으로 사용됩니다. TextBlock에서 만들어진 텍스트 줄입니다.


package {

    import flash.display.Sprite;
    import flash.text.engine.*;

    
    public class FontDescriptionExample extends Sprite {
        
        public function FontDescriptionExample():void {
            var fd:FontDescription = new FontDescription();
            fd.fontLookup = flash.text.engine.FontLookup.DEVICE;
            fd.fontName = "Palatino";
            fd.fontWeight = flash.text.engine.FontWeight.BOLD;
            fd.fontPosture = flash.text.engine.FontPosture.ITALIC;

            var ef1:ElementFormat = new ElementFormat(fd);
            ef1.fontSize = 30;
            ef1.color = 0xFF0000;
            
            var str:String = "This is flash text 0123456789";
            var tb:TextBlock = new TextBlock();
            var te1:TextElement = new TextElement(str, ef1);
            tb.content = te1;
            var line1:TextLine = tb.createTextLine(null, 600);
            addChild(line1);
            
        }
    }    
}
