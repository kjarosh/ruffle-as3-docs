HTMLLoader 클래스는 HTML 내용의 컨테이너인 표시 객체의 유형을 정의합니다.

AIR 프로파일 지원: 이 기능은 모든 데스크톱 운영 체제에서 지원되지만 휴대 장치나 AIR for TV 장치에서는 지원되지 않습니다. HTMLLoader.isSupported 속성을 사용하여 런타임에 지원을 테스트할 수 있습니다. 여러 프로파일 간 API 지원에 대한 자세한 내용은 AIR 프로파일 지원을 참조하십시오.

데스크톱 컴퓨터(데스크톱 및 확장 데스크톱 프로파일 내)에서 HTMLLoader 클래스는 내부 AIR WebKit 엔진을 사용합니다. 사용 가능한 기능 및 렌더링 모양은 StageWebView 클래스와 동일하며, ActionScript와 JavaScript 간의 긴밀한 통합 및 스크립트 브리징이 포함됩니다. StageWebView 클래스는 Flash Player 플러그인이 제공하는 시스템 웹 컨트롤을 사용하므로 StageWebView 인스턴스와 HTMLLoader 인스턴스를 동시에 사용하는 것은 좋지 않습니다. 정의되지 않은 비헤이비어가 발생하고 응용 프로그램이 종료될 수도 있기 때문입니다.

HTMLLoader의 기본 크기는 0 x 0 픽셀입니다. HTMLLoader 객체가 표시되도록 하려면 width 및 height 속성을 설정하십시오.

HTMLLoader 객체의 최대 크기는 높이 4,096픽셀 및 폭 4,096픽셀입니다. HTMLLoader 객체의 width 또는 height를 4,095보다 큰 숫자로 설정하면 HTMLLoader 객체에서 ArgumentError 예외가 발생합니다. 참고로, AIR 2 이전 버전에서는 HTMLLoader 객체의 최대 크기가 2,880픽셀이었습니다.

참고: HTML 페이지에 포함된 SWF 내용이 투명 윈도우에 표시되는 경우, transparent 또는 opaque wmode를 사용하여 내용을 포함해야 합니다. wmode의 기본값은 window이므로 값을 지정하지 않으면 SWF 내용이 표시되지 않을 수 있습니다. Windows와 Linux에서 wmode를 window 또는 opaque로 설정하면 SWF 내용은 항상 다른 내용 위에 나타납니다. 어느 wmode 설정을 사용하든 PDF 내용은 투명 윈도우에 표시할 수 없습니다.

    import flash.html.HTMLLoader;
    import flash.net.URLRequest;
    
    public class LocationChanging extends Sprite
    {
        var htmlLoader:HTMLLoader = new HTMLLoader();
        public function LocationChanging()
        {
            htmlLoader.width = stage.stageWidth;
            htmlLoader.height = stage.stageHeight;        
            htmlLoader.addEventListener( LocationChangeEvent.LOCATION_CHANGING, onLocationChanging );
            htmlLoader.load( new URLRequest( "http://www.adobe.com" ) );
            this.addChild( htmlLoader );
        }
    
        private function onLocationChanging( event:LocationChangeEvent ):void
        {
            trace( "Location changing: " + event.location );
        
            if ( event.location.indexOf( "http://www.adobe.com" ) < 0 ) 
            {
                event.preventDefault();
            }
        }
    }
