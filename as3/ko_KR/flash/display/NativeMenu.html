언어 버전: ActionScript 3.0
런타임 버전: AIR 1.0

NativeMenu 클래스에는 기본 메뉴를 정의할 수 있는 메서드 및 속성이 들어 있습니다.

AIR 프로파일 지원: 이 기능은 모든 데스크톱 운영 체제에서 지원되지만 휴대 장치나 AIR for TV 장치에서는 지원되지 않습니다. NativeMenu.isSupported 속성을 사용하여 런타임에 지원을 테스트할 수 있습니다. 여러 프로파일 간 API 지원에 대한 자세한 내용은 AIR 프로파일 지원을 참조하십시오.

기본 메뉴는 응용 프로그램이 아닌 운영 체제에 의해 제어되고 표시되는 메뉴입니다. AIR에서는 다음과 같은 기본 메뉴 유형을 지원합니다.

  • 응용 프로그램 메뉴는 OS X에서 지원됩니다. NativeApplication.supportsMenu 속성을 사용하여 호스트 운영 체제에서 응용 프로그램 메뉴가 지원되는지 여부를 테스트하십시오. 응용 프로그램 메뉴는 Mac 데스크톱 위쪽의 메뉴 모음에 표시됩니다. OS X는 모든 응용 프로그램에 대해 기본 메뉴를 제공하지만 많은 메뉴 명령이 작동하지 않습니다. 이벤트 리스너를 기본 항목에 추가하거나, 개별 메뉴 및 항목을 바꾸거나, 기본 메뉴 전체를 바꿀 수도 있습니다. NativeApplication menu 속성을 사용하여 응용 프로그램 메뉴 객체에 액세스합니다.
  • 윈도우 메뉴는 Windows와 Linux에서 지원됩니다. NativeWindow.supportsMenu 속성을 사용하여 호스트 운영 체제에서 윈도우 메뉴가 지원되는지 여부를 테스트하십시오. 윈도우 메뉴는 윈도우 제목 표시줄 아래에 표시됩니다. 메뉴가 차지하는 영역은 윈도우 스테이지의 일부가 아닙니다. 응용 프로그램은 이 영역에 표시할 수 없습니다. NativeWindow menu 속성을 사용하여 윈도우에 메뉴를 지정합니다.
  • 도크 아이콘 메뉴는 OS X에서 지원됩니다. NativeApplication.supportsDockIcon 속성을 사용하여 호스트 운영 체제에서 도크 아이콘이 지원되는지 여부를 테스트하십시오. 도크 아이콘 메뉴의 항목은 운영 체제에서 제공하는 기본 항목 위쪽에 표시됩니다. 기본 항목은 응용 프로그램 코드에서 액세스할 수 없습니다. 응용 프로그램의 DockIcon 객체의 menu 속성에 메뉴를 지정합니다.
  • 시스템 트레이 아이콘 메뉴는 Windows 및 대부분의 Linux 운영 체제에서 지원됩니다. NativeApplication.supportsSystemTrayIcon 속성을 사용하여 호스트 운영 체제에서 시스템 트레이 아이콘이 지원되는지 여부를 테스트하십시오. 시스템 트레이 아이콘 메뉴는 컨텍스트 메뉴와 마찬가지로, 아이콘을 마우스 오른쪽 버튼으로 클릭할 때 표시됩니다. 응용 프로그램의 SystemTrayIcon 객체의 menu 속성에 메뉴를 지정합니다.
  • 컨텍스트 메뉴는 모든 운영 체제에서 지원됩니다. 컨텍스트 메뉴는 응용 프로그램에 표시된 InteractiveObject를 마우스 오른쪽 버튼으로 클릭하거나 Command 키를 누른 상태에서 클릭할 때와 같이 사용자 인터페이스 이벤트가 발생할 때 표시됩니다. 메뉴를 표시하는 UI 메커니즘은 호스트 운영 체제 및 하드웨어에 따라 다릅니다. InteractiveObject의 contextMenu 속성에 메뉴를 지정합니다. AIR에서는 NativeMenu 클래스 또는 ContextMenu 클래스를 사용하여 컨텍스트 메뉴를 만들 수 있습니다. Flash Player에서는 ContextMenu 클래스만 사용할 수 있습니다. AIR의 ContextMenus에는 내장 항목이 없으므로 기본 컨텍스트 메뉴가 표시되지 않습니다.
  • 팝업 메뉴는 모든 운영 체제에서 지원됩니다. 팝업 메뉴는 컨텍스트 메뉴와 작동 방식이 동일하지만 사용자 인터페이스 이벤트가 발생할 때 표시되지 않고 메뉴 display() 메서드를 사용하여 표시됩니다. 팝업 메뉴는 다른 객체에 연결되지 않습니다. 기본 메뉴를 만든 후 display() 메서드를 호출하기만 하면 됩니다.

메뉴 객체에는 메뉴 항목이 포함됩니다. 메뉴 항목은 명령, 하위 메뉴 또는 분리 기호 선을 나타낼 수 있습니다. addItem() 또는 addItemAt() 메서드를 사용하여 메뉴에 메뉴 항목을 추가합니다. 메뉴 항목의 표시 순서는 메뉴 items 배열에 있는 항목의 순서와 일치합니다.

하위 메뉴를 만들려면 부모 메뉴 객체에 메뉴 항목을 추가합니다. 부모 메뉴에서 일치하는 메뉴 항목의 submenu 속성에 하위 메뉴를 나타내는 메뉴 객체를 지정합니다.

참고: 윈도우의 루트 메뉴 및 응용 프로그램 메뉴에는 하위 메뉴 항목만 들어 있어야 합니다. 이러한 유형의 메뉴에 대한 사용자의 예상과는 달리 하위 메뉴를 나타내지 않는 항목은 표시되지 않을 수 있습니다.

메뉴의 명령 항목 또는 이 항목의 하위 메뉴 중 하나를 선택할 때 메뉴는 select 이벤트를 전달합니다. 하위 메뉴 및 분리 기호 항목은 선택할 수 없습니다. 이벤트 객체의 target 속성은 선택한 항목을 참조합니다.

메뉴 항목 중 하나에 해당하는 키가 눌리면 메뉴가 표시되기 직전에 메뉴가 preparing 이벤트를 전달합니다. 이 이벤트를 사용하면 응용 프로그램의 현재 상태를 기준으로 메뉴의 내용을 업데이트할 수 있습니다.

참고: Flex 프레임워크를 사용하는 경우에는 FlexNativeMenu 클래스를 사용하는 것을 고려합니다. ActionScript 코드를 작성하여 항목별로 메뉴 구조를 만드는 것보다 MXML에서 선언적으로 메뉴를 정의하는 것이 일반적으로 더 간편합니다.

    addItem

()메서드
public function addItem(item:NativeMenuItem):NativeMenuItem

언어 버전: ActionScript 3.0
런타임 버전: AIR 1.0

메뉴의 맨 아래에 메뉴 항목을 추가합니다.

컨텍스트 메뉴를 만들 때 NativeMenuItem 또는 ContextMenuItem 객체를 추가할 수 있습니다. 그러나 메뉴의 모든 항목이 같은 속성을 가지도록 컨텍스트 메뉴에서 하나의 객체 유형만 사용하는 것이 좋습니다.

참고: 항목의 하위 메뉴를 해당 메뉴 자체로 설정할 경우 순환 참조가 발생하여 항목을 메뉴에 추가할 때 응용 프로그램이 중지될 수 있습니다.

매개 변수

item:NativeMenuItem — 메뉴의 맨 아래에 추가할 NativeMenuItem 객체입니다.

반환값
NativeMenuItem

오류
ArgumentError — item이 null인 경우입니다.
