Предоставляет доступ к специфичному для адреса назначения коду ошибки. Для получения дополнительной подробной информации см. описание свойств faultString и faultDetails.

Формат кодов ошибок предоставляется удаленным адресатом, но, как правило, имеет следующий вид: host.operation.error или Channel.operation.error Например, Server.Connect.Failed и Channel.Connect.Failed.

Channel.Connect.Failed вызывается классом Channel и его подклассами (RTMPChannel, AMFChannel, HTTPChannel и т. д.) всякий раз, когда возникает проблема при попытке подключения канала к удаленному адресу назначения. Channel.Call.Failed вызывается AMFChannel, когда канал уже подключен, но получает код Call.Failed из основного NetConnection. Только класс AMFChannel прослушивает NetConnection.Call.Failed, преобразуемый в Channel.Call.Failed.

ChannelFaultEvent

()Конструктор
public function ChannelFaultEvent(type:String, bubbles:Boolean = false, cancelable:Boolean = false, channel:Channel = null, reconnecting:Boolean = false, code:String = null, level:String = null, description:String = null, rejected:Boolean = false, connected:Boolean = false)

Язык версии: ActionScript 3.0
Версии продуктов: BlazeDS 4, Adobe Digital Enterprise Platform Data Services for Java EE 3
Версии среды выполнения: Flash Player 9, AIR 1.1

Создает экземпляр этого события с указанным типом. Учтите, что аргументы rejected и connected, которые соответствуют свойствам, определенным суперклассом ChannelEvent, первоначально не были включены в подпись этого метода. Они были добавлены в конец списка аргументов для обеспечения обратной совместимости даже в случае отличия данной подписи от подписи конструктора ChannelEvent.

Параметры
type:String — Тип события.
 
bubbles:Boolean (default = false) — Указывает, может ли событие передаваться вверх по иерархии списка отображения.
 
cancelable:Boolean (default = false) — Указывает, можно ли предотвратить поведение, связанное с событием.
 
channel:Channel (default = null) — Канал, генерирующий событие.
 
reconnecting:Boolean (default = false) — Указывает, находится канал в процессе повторного подключения или нет.
 
code:String (default = null) — Код ошибки.
 
level:String (default = null) — Уровень ошибки.
 
description:String (default = null) — Описание ошибки.
 
rejected:Boolean (default = false) — Указывает, было ли отклонено подключение канала с подавлением автоматического повторного подключения.
 
connected:Boolean (default = false) — Указывает, подключен ли канал, сгенерировавший это событие.

[ X ]Почему по-английски?
Содержимое Справочника ActionScript 3.0 отображается на английском языке

Не все части Справочника ActionScript 3.0 переводятся на все языки. Если какой-то текстовый элемент не переведен, он отображается на английском языке. Например, компонент ga.controls.HelpBox не переведен ни на один из языков. Это значит, что в русской версии справки компонент ga.controls.HelpBox будет отображаться на английском языке.
