El siguiente es un subconjunto preciso de las etiquetas XML de texto temporizado admitidas por el componente FLVPlaybackCaptioning.

etiqueta tt - etiqueta de nivel superior

Se omiten todos los atributos de la etiqueta tt.

Se permite la etiqueta metadata, pero se omite en cualquier nivel del documento.

Se puede utilizar la etiqueta set pero se omite en cualquier nivel del documento.

Se admiten los siguientes valores de tiempo:

  • formato de reloj completo

    (p.ej. 00:03:00.1)

  • formato de reloj parcial

    (p.ej. 03:00.1)

  • tiempo de desfase

    (p.ej. 100.1s o 20h)Las horas locales convertidas sin unidades (por ejemplo, 10) se asumen como segundos.

    • No se admite ningún formato de hora que incluya fotogramas o marcas (por ejemplo: 00:03:00:05, 00:03:00:05.1, 30f, 30t).

    • begin, dur y end se admiten solo en la etiqueta p; por lo tanto, todas las horas proporcionadas son relativas a los medios.

Los atributos xml:lang se omiten.

Los atributos xml:space se omiten, el componente FLVPlaybackCaptioning asume xml:space=default.

La etiqueta head es opcional (use una o ninguna).

  • Las etiquetas de estilos requieren un atributo id; admite una o varias etiquetas de estilos dentro.

    • Atributos admitidos:

      • tts:backgroundColor

        Toda el área de texto tiene el mismo color de fondo. Puede establecerlo en cualquier nivel de etiqueta admitido, pero afectará a cada párrafo, selección, etc., del texto visible actual.

        Compatibilidad con todas las representaciones de color válidas, aunque el componente alfa se omite si no es 0. Si el valor alfa es 0, no se muestra ningún color de fondo. Un valor de transparent se trata como un valor 0 de alfa.

      • tts:color

        Compatibilidad con todas las representaciones de color válidas, aunque el componente alfa se omite. Un valor de transparent se trata como negro.

      • tts:fontFamily

      • tts:fontSize

        Compatibilidad con el primer valor (vertical) solo (si existen atributos fontSize). Las unidades son opcionales (se asume que se expresan en píxeles). Compatibilidad con unidades absolutas (por ejemplo, 12) y relativas (por ejemplo, +2). No se admiten los valores de porcentaje.

      • tts:fontWeight

      • tts:textAlign

      • tts:wrapOption

        El atributo wrapOption se aplica a toda la zona de texto (por ejemplo, cada párrafo, selección, etc., visible actualmente.

    • Atributos no admitidos:

      • tts:direction

      • tts:display

      • tts:displayAlign

      • tts:dynamicFlow

      • tts:opacity

      • tts:origin

      • tts:overflow

      • tts:padding

      • tts:showBackground

      • tts:textOutline

      • tts:unicodeBidi

      • tts:visibility

      • tts:writingMode

      • tts:zIndex

Se omite la etiqueta layout, incluidas las posibles etiquetas region que contenga.

Se requiere la tarjeta body; solo se permite una etiqueta body.

  • La etiqueta body admite el atributo style y cualquier atributo de estilos.

  • Los atributos begin, dur, end, id, region, timeContainer, etc., se omiten en la etiqueta div.

  • Compatibilidad con cero o más etiquetas p.

    • Compatibilidad con los atributos begin, dur, style y cualquier atributo de estilos de la etiqueta p.

    • Los atributos id, region, timeContainer, etc. se omiten en la etiqueta p.

    • El atributo begin es necesario. Si se omite el atributo begin, se produce un error sin mensaje.

    • Si no hay ningún atributo dur ni end, se descarta el texto en favor del siguiente.

    • Si se encuentran ambos atributos dur y end, se utiliza el atributo end.

    • La etiqueta p admite una gama de etiquetas span, br y text (de acuerdo con lo estipulado en la especificación W3C).

    • Se admiten las etiquetas metadata y set, aunque se omiten.

    • La etiqueta span

      -No se admiten las etiquetas span anidadas.

      -Compatibilidad con los atributos de style y con los atributos de estilos de la etiqueta span.

      -En una etiqueta span se omiten los atributos begin, dur, end, id, region, timeContainer, etc.

    • La etiqueta br. Se omiten todos los atributos y el contenido de una etiqueta br.

Cuando el componente FLVPlaybackCaptioning analiza el archivo Timed Text XML, el principio de la información de tiempo de texto se transforma en puntos de referencia de ActionScript y se añaden a la lista de instancias de FLVPlayback. La convención de asignación de nombres de los puntos de referencia utiliza el prefijo inicial "fl.video.caption2.0." No cree puntos de referencia con estos nombres de prefijo a menos que desee utilizarlos para texto.

[ X ]¿Por qué hay texto en inglés?
Cierto contenido de la Referencia de ActionScript 3.0 se muestra en inglés

No todo el contenido de la Referencia de ActionScript 3.0 se traduce a todos los idiomas. Si un elemento del lenguaje no se traduce, aparecerá en inglés. Por ejemplo, la clase ga.controls.HelpBox no está traducida en ningún idioma. Por lo tanto, en la versión en español de la referencia, la clase ga.controls.HelpBox aparecerá en inglés.
