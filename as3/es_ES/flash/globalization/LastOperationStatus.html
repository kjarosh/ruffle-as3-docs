Paqueteflash.globalization
Clasepublic final class LastOperationStatus
HerenciaLastOperationStatus Object

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: Flash Player 10.1, AIR 2

La clase LastOperationStatus enumera valores de constantes que representan el estado de la última operación de servicio de globalización. Estos valores se pueden recuperar a través de la propiedad de sólo lectura lastOperationStatus, disponible en la mayoría de clases de globalización.

ILLEGAL_ARGUMENT_ERROR

Constante 
public static const ILLEGAL_ARGUMENT_ERROR:String = "illegalArgumentError"

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: Flash Player 10.1, AIR 2

Indica que un argumento transferido al método no es válido.

Por ejemplo, el código siguiente muestra que se establece un estado de error de argumento no cuando la propiedad CurrencyFormatter.grouping se establece con el valor no válido "3;". 

        var cf:CurrencyFormatter = new CurrencyFormatter("en-US");
        cf.groupingPattern = "3;";
        trace(cf.lastOperationStatus); // "illegalArgumentError"

PARSE_ERROR

Constante 
public static const PARSE_ERROR:String = "parseError"

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: Flash Player 10.1, AIR 2

Indica que ha fallado el análisis un número. Este estado puede ser devuelto mediante métodos de la análisis de las clases formatter, como CurrencyFormatter.parse() y NumberFormatter.parseNumber(). Por ejemplo, si el valor "12abc34" se transfiere como parámetro al método CurrencyFormatter.parse() método, el valor devuelto será "NaN" y el valor lastOperationStatus se establecerá como LastOperationStatus.PARSE_ERROR.

PATTERN_SYNTAX_ERROR

Constante 
public static const PATTERN_SYNTAX_ERROR:String = "patternSyntaxError"

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: Flash Player 10.1, AIR 2

Indica que el patrón para dar formato a un número, fecha u hora no es válido. Este estado se establece cuando el sistema operativo del usuario no admite el patrón determinado.

Por ejemplo, el código siguiente muestra el valor de la lastOperationStatus tras haberse utilizado un patrón "xx" no válido para dar formato a la fecha:

        var df:DateTimeFormatter = new DateTimeFormatter("en-US");
        df.setDateTimePattern("xx");
        trace(df.lastOperationStatus); // "patternSyntaxError"
