Körningsmiljöversioner: Flash Player 9.0.28.0, AIR 1.0

Klassen UIComponent är basklassen för alla visuella komponenter, både interaktiva och icke-interaktiva. Interaktiva komponenter definieras som komponenter som tar emot indata som tangentbords- eller musaktivitet. Icke-interaktiva komponenter används för att visa data; de svarar inte på användarinteraktion. Komponenterna ProgressBar och UILoader är exempel på icke-interaktiva komponenter.

Med tabb- och piltangenter kan fokus flyttas till och över en interaktiv komponent; en interaktiv komponent kan acceptera händelser på låg nivå som inmatning från tangentbord- och musenhet. En interaktiv komponent kan också inaktiveras så att den inte kan ta emot inmatning från tangentbord- och musenhet.

getStyle

public function getStyle(style:String):Object

Språkversion: ActionScript 3.0
Produktversion: Flash CS3
Körningsmiljöversioner: Flash Player 9.0.28.0, AIR 1.0

Hämtar en formategenskap som är inställd i komponentens sökkedja.

Den type som den här metoden returnerar varierar beroende på den formategenskap som metoden hämtar. Intervallet av möjliga types inkluderar Boolean; String; Number; int; uint för en RGB-färg; en Class för ett skal; eller valfritt objekt.

Om du anropar den här metoden för att hämta en viss formategenskap kommer det att vara av en känd typ som du kan lagra i en variabel av samma typ. Datatypsbyte behövs inte. Istället räcker det med en enkel uppdragssats som den här:

var backgroundColor:uint = getStyle("backgroundColor");

Om formategenskapen inte har ställts in någonstans i formatsökkedjan returnerar den här metoden värdetundefined. Observera att undefined är ett eget värde som inte är detsamma som false, "", NaN, 0, eller null. Giltiga formatvärden är aldrig undefined. Det går att använda den statiska metoden StyleManager.isValidStyleValue() om du vill testa om ett värde har ställts in.

Parametrar

style:String — Namnet på formategenskapen.

Returnerar
Object — Formatvärde.

Exempel  ( Så här använder du exemplet )

I följande exempel skapas en DataGrid-komponentinstans som visar standardformat och aktuella format för en ComboBox-komponent: 

import fl.controls.ComboBox;
import fl.controls.DataGrid;
import fl.controls.dataGridClasses.DataGridColumn;
import fl.data.DataProvider;

var myComboBox:ComboBox = new ComboBox();
myComboBox.setStyle("textPadding", 6);
myComboBox.setStyle("disabledAlpha", 0.8);

var styleObject:Object = ComboBox.getStyleDefinition();
var styleDP:DataProvider = new DataProvider();
var styleName:String;
var styleDefaultValue:String;
var styleCurrentValue:String;

for (styleName in styleObject) {
    styleDefaultValue = String(styleObject[styleName]);
    styleCurrentValue = String(myComboBox.getStyle(styleName));
    styleDP.addItem({name:styleName, defaultValue:styleDefaultValue, currentValue:styleCurrentValue});
}
styleDP.sortOn("name", Array.CASEINSENSITIVE);

var c1:DataGridColumn = new DataGridColumn("name");
c1.headerText = "Style name:";
var c2:DataGridColumn = new DataGridColumn("defaultValue");
c2.headerText = "Default style value:";
var c3:DataGridColumn = new DataGridColumn("currentValue");
c3.headerText = "Current style value:"

var styleDataGrid:DataGrid = new DataGrid();
styleDataGrid.addColumn(c1); // name
styleDataGrid.addColumn(c2); // defaultValue
styleDataGrid.addColumn(c3); // currentValue
styleDataGrid.dataProvider = styleDP;
styleDataGrid.setSize(stage.stageWidth - 20, stage.stageHeight - 20);
styleDataGrid.move(10, 10);
addChild(styleDataGrid);

I följande exempel skapas en Button-komponent som växlar synlighet för en List-komponent: 

import fl.controls.Button;

var toggleButton:Button = new Button();
toggleButton.label = "toggle visibility";
toggleButton.toggle = true;

myList.addEventListener(ListEvent.ITEM_ROLL_OUT, itemRollOverHandler);

    myList.addEventListener(ListEvent.ITEM_ROLL_OUT, itemRollOverHandler);
