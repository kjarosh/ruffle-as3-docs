Klassen TextLayoutFormat omfattar alla egenskaper för textlayout. Dessa egenskaper påverkar format och stil i ett textflöde på behållarnivå, styckenivå och textnivå. Både klassen ContainerController och FlowElement har format-egenskaper, som du kan använda för att koppla en TextLayoutFormat-instans till dem. Tilldela en behållare ett TextLayoutFormat-objekt för att påverka formatet för allt innehåll i behållaren. Koppla ett TextLayoutFormat-objekt till en FlowElement-underordnad för att ange formatering för det specifika elementet: TextFlow, ParagraphElement, DivElement, SpanElement, InlineGraphicElement, LinkElement och TCYElement. Förutom egenskapen format definierar dessa klasser också varje individuell TextLayoutFormat-egenskap så att du kan åsidosätta inställningen för en viss formategenskap för detta element.

Eftersom du kan ange ett visst format på flera nivåer, kan konflikter uppstå. Du kan t.ex. ange färgen svart för texten på TextFlow-nivån medan ett SpanElement-objekt ställer in färgen blå. Den allmänna regeln är att inställningen på den lägsta nivån i textflödet har högre prioritet. Om ligaturnivån har ställts in för en TextFlow-instans men också för ett DivElement-objekt, kommer DivElement-inställningen att prioriteras.

Överlappande format hänvisar till att hämta format från en högre nivå i textflödet om ett formatvärde inte har definierats på en lägre nivå. Om ett format inte har definierats på ett element när det ska återges, hämtas antingen standardvärdet eller också överlappas värdet eller tas från värdet för ett överordnat element. Om genomskinligheten (egenskapen textAlpha) för texten inte har definierats på ett SpanElement-objekt, men har angetts för TextFlow, överlappas värdet för egenskapen TextFlow.textAlpha till SpanElement-objektet och tillämpas på texten för detta intervall. Resultatet av överlappningen, eller summan av de format som används på elementet, sparas i elementets computedFormat-egenskap.

På samma sätt kan du tillämpa användarformat med egenskapen userStyles i klasserna ContainerController och FlowElement. Med den här egenskapen kan du läsa eller skriva ett lexikon med användarformat och tillämpa inställningarna på en behållare eller ett textflödeselement. Lexikonet är ett objekt som består av stylename-value-par. Format angivna av egenskapen userStyles prioriteras före alla andra.

De flesta format som är odefinierade ärver värdet på det direkt överordnade objektet under en överlappning. Det finns dock ett mindre antal format, som inte ärver det överordnade objektets värde, utan i stället får ett eget standardvärde.

Formategenskaper som får egna standardvärden, om de inte definieras, omfattar följande: backgroundAlpha, backgroundColor, columnCount, columnGap, columnWidth, lineBreak, paddingBottom, paddingLeft, paddingRight, paddingTop och verticalAlign

alignmentBaseline

egenskap
alignmentBaseline:*

Språkversion: ActionScript 3.0
Körningsmiljöversioner: Flash Player 10, AIR 1.5

Anger den baslinje efter vilken den dominanta baslinjen justeras. Om du till exempel anger dominantBaseline som ASCENT och anger alignmentBaseline som DESCENT justeras textens överkant mot DESCENT-baslinjen eller under raden. Det största elementet på raden styr oftast baslinjen.

Giltiga värden är TextBaseline.ROMAN, TextBaseline.ASCENT, TextBaseline.DESCENT, TextBaseline.IDEOGRAPHIC_TOP, TextBaseline.IDEOGRAPHIC_CENTER, TextBaseline.IDEOGRAPHIC_BOTTOM, TextBaseline.USE_DOMINANT_BASELINE och FormatValue.INHERIT.

Standardvärdet är odefinierat, vilket anger att det inte ställts in.

Om värdet är odefinierat under överlappning ärver den här egenskapen sitt värde från ett överordnat objekt. Om inget överordnat objekt har den här egenskapen, får den värdet för TextBaseline.USE_DOMINANT_BASELINE.

Exempel  ( Så här använder du exemplet )
I det här exemplet får alignmentBaseline tre olika värden – USE_DOMINANT_BASELINE, DESCENT och IDEOGRAPHIC_CENTER – för tre omfång i ett stycke där dominantBaseline är inställt på ASCENT. 
package flashx.textLayout.formats.examples {
    import flash.display.Sprite;
    import flash.text.engine.TextBaseline;
    import flashx.textLayout.container.ContainerController;
    import flashx.textLayout.elements.ParagraphElement;
    import flashx.textLayout.elements.SpanElement;
    import flashx.textLayout.elements.TextFlow;
    import flashx.textLayout.formats.TextAlign;

    public class TextLayoutFormat_alignmentBaselineExample extends Sprite
    {
        public function TextLayoutFormat_alignmentBaselineExample()
        {
            // create text flow with dominantBaseline of ASCENT and 3 spans
            var textFlow:TextFlow = new TextFlow();
            var paragraph:ParagraphElement = new ParagraphElement();
            paragraph.fontFamily = "Helvetica";
            paragraph.fontSize = 20;
            paragraph.dominantBaseline = TextBaseline.ASCENT    
            var spanOne:SpanElement = new SpanElement();
            var spanTwo:SpanElement = new SpanElement();
            var spanThree:SpanElement = new SpanElement();
            spanOne.text = "Hello, World  ";
            spanTwo.text = "Hello, World  ";
            spanThree.text = "Hello, World";
            
            // add spans to paragraph and specify alignment baseline for each
            paragraph.addChild(spanOne);
            spanOne.alignmentBaseline = TextBaseline.USE_DOMINANT_BASELINE;    
            paragraph.addChild(spanTwo);
            spanTwo.alignmentBaseline = TextBaseline.DESCENT;
            paragraph.addChild(spanThree);
            spanThree.alignmentBaseline = TextBaseline.IDEOGRAPHIC_CENTER;
            
            // add paragraph to TextFlow and set container position
            textFlow.addChild(paragraph);
            this.x = 50;
            this.y = 100;
            
            // update controller to display on stage
            textFlow.flowComposer.addController(new ContainerController(this,400,300));
            textFlow.flowComposer.updateAllControllers();    
        }
    }
}

breakOpportunity:*

Språkversion: ActionScript 3.0
Körningsmiljöversioner: Flash Player 10, AIR 1.5

Styr var rader tillåts att radbrytas när text måste radbrytas över flera rader. Ange som BreakOpportunity.AUTO om du vill radbryta texten normalt. Ange som BreakOpportunity.NONE om du inte vill radbryta texten, såvida den inte flödar över och det inte finns några andra ställen att radbryta raden. Ange som BreakOpportunity.ANY om raden kan radbrytas var som helst, och inte bara mellan ord. Ange som BreakOpportunity.ALL om varje typografisk grupp ska placeras på en egen rad (praktiskt för text på en bana).

Giltiga värden är BreakOpportunity.ALL, BreakOpportunity.ANY, BreakOpportunity.AUTO, BreakOpportunity.NONE och FormatValue.INHERIT.

Standardvärdet är odefinierat, vilket anger att det inte ställts in.

Om värdet är odefinierat under överlappning ärver den här egenskapen sitt värde från ett överordnat objekt. Om inget överordnat objekt har den här egenskapen får den värdet för BreakOpportunity.AUTO.
