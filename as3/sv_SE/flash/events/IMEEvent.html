Klasspublic class IMEEvent
ArvIMEEvent TextEvent Event Object

Språkversion: ActionScript 3.0
Körningsmiljöversioner: AIR 1.0, Flash Player 9, Flash Lite 4

Ett IMEEvent-objekt skickas när användaren anger text med IME (input method editor). IME används vanligen för att ange text från språk med ideogram, tecken, istället för bokstäver, exempelvis japanska, kinesiska eller koreanska. Det finns två IME-händelser: IMEEvent.IME_COMPOSITION och IMEEvent.IME_START_COMPOSITION.

Konstruktordetaljer

IMEEvent

()Konstruktor
public function IMEEvent(type:String, bubbles:Boolean = false, cancelable:Boolean = false, text:String = "", imeClient:IIMEClient = null)

Språkversion: ActionScript 3.0
Körningsmiljöversioner: AIR 1.0, Flash Player 9, Flash Lite 4

Skapar ett nytt Event-objekt med information som är relevant för IME-händelser. Event-objekt skickas som parametrar till händelseavlyssnare.

Parametrar
type:String — Händelsens typ. Händelseavlyssnare kan hämta informationen genom den ärvda egenskapen type. Det finns bara en IME-händelse: IMEEvent.IME_COMPOSITION.
 
bubbles:Boolean (default = false) — Bestämmer om Event-objektet deltar i bubblingssteget av händelseflödet. Händelseavlyssnare kan hämta informationen genom den ärvda egenskapen bubbles.
 
cancelable:Boolean (default = false) — Avgör om Event-objektet kan avbrytas. Händelseavlyssnare kan hämta informationen genom den ärvda egenskapen cancelable.
 
text:String (default = "") — Lässträngen från IME. Det är den första strängen som användaren skriver, innan någon kandidat har valts. Den slutgiltiga strängen levereras till objektet med tangentbordsfokus i en TextEvent.TEXT_INPUT-händelse. Händelseavlyssnare kan hämta informationen genom den ärvda egenskapen text.
 
imeClient:IIMEClient (default = null) — En uppsättning återkopplingar som används av textmotorn för att kommunicera med IME. Det är praktiskt om koden har en egen textmotor och själv återger textrader i stället för att TextField-objekt eller TextLayoutFramework används.

Språkversion: ActionScript 3.0
Körningsmiljöversioner: AIR 1.0, Flash Player 10.1

För att hantera IME-textindata måste fältet imeClient för händelsen hos mottagaren anges till ett objekt som implementerar IIMEClient-gränssnittet. Om imeClient inte anges används IME-disposition som inte är infogad i stället, och den slutliga dispositionen skickas som en TEXT_INPUT-händelse.
