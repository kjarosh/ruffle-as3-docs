Beginnend mit Flex 4.0 empfiehlt Adobe die Verwendung der Klasse spark.components.ComboBox als Alternative zu dieser Klasse.
Beginnend mit Flex 4.0 empfiehlt Adobe die Verwendung der Klasse spark.components.DropDownList als Alternative zu dieser Klasse.

Das ComboBox-Steuerelement umfasst eine Dropdownliste, aus welcher der Benutzer einen einzelnen Wert auswählen kann. Seine Funktion ist der des SELECT-Elements in HTML sehr ähnlich. Das ComboBox-Steuerelement kann bearbeitbar sein; in diesem Fall kann der Benutzer Text, der nicht in der Liste steht, in den TextInput-Teil des Kombinationsfelds eingeben.

Das ComboBox-Steuerelement verfügt über die folgenden Standardgrößeneigenschaften:

MerkmaleBeschreibung
StandardgrößeBreit genug für den längsten Eintrag in der Dropdownliste im Anzeigebereich des Hauptsteuerelements zuzüglich Dropdownschaltfläche. Ist die Dropdownliste nicht sichtbar, so beruht die Standardhöhe auf der Textgröße der Beschriftung.

Die Standardhöhe der Dropdownliste beträgt fünf Zeilen oder die Anzahl der Einträge in der Dropdownliste, je nachdem, welcher Wert geringer ist. Die Standardhöhe jedes Eintrags in der Dropdownliste beträgt 22 Pixel.

Maximale Größe5000 x 5000.

Die data-Eigenschaft erlaubt es Ihnen, einen Wert an die Komponente zu übergeben, wenn Sie diese in einem Elementrenderer oder -editor verwenden. Normalerweise verwenden Sie Datenbindung, um ein Feld der data-Eigenschaft an eine Eigenschaft dieser Komponente zu binden.

Das ComboBox-Steuerelement verwendet die listData- und data-Eigenschaften wie folgt: Befindet sich die ComboBox in einem DataGrid-Steuerelement, so erwartet sie, dass die dataField-Eigenschaft der Spalte einer Eigenschaft in den Daten zugeordnet ist und setzt selectedItem auf diese Eigenschaft. Befindet sich die ComboBox in einem List-Steuerelement, so erwartet sie, dass die labelField-Eigenschaft der Liste einer Eigenschaft in „data“ zugeordnet ist und setzt selectedItem auf diese Eigenschaft. Andernfalls wird selectedItem auf „data“ gesetzt.

Sie stellen diese Eigenschaft nicht in MXML ein.

Diese Eigenschaft kann als Quelle für die Datenbindung verwendet werden. Beim Ändern dieser Eigenschaft wird das dataChange -Ereignis ausgelöst.

Als Beispiel kann ein Datensatz angeführt werden, der die Felder lastName und firstName besitzt. Sie möchten aber die vollständigen Namen anzeigen. Mit labelFunction geben Sie eine Rückruffunktion an, welche die entsprechenden Felder verwendet und einen anzeigbaren String zurückgibt.

labelFunction verwendet ein einziges Argument (das Element im dataProvider) und gibt einen String zurück:

      myLabelFunction(item:Object):String

Diese Eigenschaft kann als Quelle für die Datenbindung verwendet werden. Beim Ändern dieser Eigenschaft wird das labelFunctionChanged -Ereignis ausgelöst.
