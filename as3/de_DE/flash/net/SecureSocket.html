Sprachversion: ActionScript 3.0
Laufzeitversionen: Flash Player 11, AIR 2

Die SecureSocket-Klasse ermöglicht es dem Code, Socketverbindungen unter Verwendung der Protokolle Secure Sockets Layer (SSL) und Transport Layer Security herzustellen.

Unterstützung von AIR-Profilen: Diese Funktion wird auf allen Desktopbetriebssystemen unterstützt, jedoch nicht auf allen Geräten mit AIR für TV. Auf mobilen Geräten wird es unter Android und iOS, angefangen bei AIR 20 unterstützt. Mit der SecureSocket.isSupported-Eigenschaft können Sie testen, ob zur Laufzeit Unterstützung gegeben ist. Weitere Informationen zum API-Support über mehrere Profile finden Sie unter Unterstützung von AIR-Profilen.

Die SSL/TLS-Protokolle stellen einen Mechanismus bereit, um beide Aspekte einer sicheren Socketverbindung zu verarbeiten:

  1. Verschlüsselung der Datenkommunikation, die über das Socket läuft
  2. Authentifizierung der Hostidentität anhand dessen Zertifikats

Die unterstützten Verschlüsselungsprotokolle sind SSL 3.1 und höher sowie TLS 1.0 und höher. (TLS ist das Nachfolgerprotokoll von SSL. TLS 1.0 entspricht SSL 3.1, TLS 1.1 entspricht SSL 3.2 usw.) SSL-Versionen 3.0 oder niedriger werden nicht unterstützt.

Die Validierung des Serverzertifikats wird mithilfe des Vertrauensspeichers und der Zertifikatvalidierungsunterstützung der Clientplattform ausgeführt. Darüber hinaus können Sie Ihre eigenen Zertifikate programmatisch mit der addBinaryChainBuildingCertificate()-Methode hinzufügen. Diese API wird derzeit nicht auf iOS unterstützt. Bei der Verwendung dieser API auf iOS wurde eine Ausnahme auftreten: ArgumentError: Error #2004

Die SecureSocket-Klasse stellt nur zu Servern mit gültigen, vertrauenswürdigen Zertifikaten eine Verbindung her. Wenn es ein Problem mit dem Zertifikat eines Servers gibt, können Sie keine Verbindung zu diesem Server herstellen. Es ist zum Beispiel nicht möglich, eine Verbindung zu einem Server herzustellen, dessen Zertifikat abgelaufen ist. Dasselbe gilt für ein Zertifikat, das nicht mit einem vertrauenswürdigen Ankerzertifikat verkettet ist. Die Verbindung wird nicht hergestellt, selbst wenn das Zertifikat ansonsten gültig ist.

Die SecureSocket-Klasse ist für die verschlüsselte Kommunikation mit einem vertrauenswürdigen Server hilfreich. Ansonsten verhält sich ein SecureSocket-Objekt wie ein normales Socket-Objekt.

Um die SecureSocket-Klasse zu verwenden, erstellen Sie ein SecureSocket-Objekt (new SecureSocket()). Als Nächstes richten Sie Ihre Listener ein und führen dann SecureSocket.connect(host, port) aus. Nachdem Sie die Verbindung zum Server erfolgreich hergestellt haben, setzt das Socket ein connect-Ereignis ab. Eine Verbindung ist dann erfolgreich, wenn die Sicherheitsprotokolle des Servers unterstützt werden und seine Zertifikate gültig und vertrauenswürdig sind. Wenn das Zertifikat nicht validiert werden kann, setzt der Socket ein IOError-Ereignis ab.

Wichtig: Das Online Certificate Status Protocol (OCSP) wird nicht von allen Betriebssystemen unterstützt. Benutzer können die OCSP-Überprüfung auf einzelnen Computern deaktivieren. Wenn OCSP nicht unterstützt wird oder deaktiviert ist und ein Zertifikat nicht die erforderlichen Informationen zur Überprüfung der Zertifikatsperrung anhand einer Zertifikatsperrliste (Certificate Revocation List, CRL) enthält, wird die Zertifikatsperrung nicht überprüft. Wenn das Zertifikat ansonsten gültig ist, wird es akzeptiert. Dieses Szenario lässt es zu, dass ein Server ein gesperrtes Zertifikat verwendet.

serverCertificateStatus

Eigenschaft 
serverCertificateStatus:String  [schreibgeschützt]

Sprachversion: ActionScript 3.0
Laufzeitversionen: Flash Player 11, AIR 2

Gibt den Status des Serverzertifikats zurück.

Der Status ist CertificateStatus.UNKNOWN, bis der Socket versucht, eine Verbindung zu einem Server herzustellen. Nach der Validierung wird als Status einer der von der CertificateStatus-Klasse aufgezählten Strings angegeben. Die Verbindung wird nur dann erfolgreich hergestellt, wenn das Zertifikat gültig und vertrauenswürdig ist. Deshalb ist nach einem connect-Ereignis der Wert von serverCertificateStatus immer trusted.

Hinweis: Nachdem das Zertifikat erfolgreich validiert oder abgelehnt wurde, wird der Status bis zum nächsten Aufruf der connect()-Methode nicht mehr aktualisiert. Durch den Aufruf von close() wird der Status nicht auf „unknown“ (unbekannt) zurückgesetzt.

Methodendetails

addBinaryChainBuildingCertificate

()Methode
public function addBinaryChainBuildingCertificate(certificate:ByteArray, trusted:Boolean):void

Sprachversion: ActionScript 3.0
Laufzeitversionen: Flash Player 11, AIR 3

Fügt der lokalen Zertifikatkette, die Ihr System zum Überprüfen des Serverzertifikats verwendet, ein X.509-Zertifikat hinzu. Das Zertifikat ist zeitlich befristet und gilt für die Dauer der Sitzung.

Die Überprüfung des Serverzertifikats beruht auf dem Vertrauensspeicher Ihres Systems zur Zertifikatverkettung und -überprüfung. Verwenden Sie diese Methode, um Zertifikatketten und vertrauenswürdige Anker programmgesteuert hinzuzufügen.

Unter Mac OS ist das Schlüsselbund des Systems das standardmäßige Schlüsselbund, das beim SSL/TLS-Handshakeprozess verwendet wird. Alle Zwischenzertifikate in diesem Schlüsselbund werden beim Erstellen der Zertifikatkette einbezogen.

Das Zertifikat, das Sie mit dieser API hinzufügen, muss ein DER-kodiertes X.509-Zertifikat sein. Wenn der trusted-Parameter den Wert „true“ hat, gilt das Zertifikat, das Sie mit dieser API hinzufügen, als vertrauenswürdiger Anker.

Weitere Informationen zu X.509-Zertifikaten finden Sie unter RFC2459.

Parameter

certificate:ByteArray — Ein ByteArray-Objekt, das ein digitales DER-kodiertes X.509-Zertifikat enthält.
 
trusted:Boolean — Setzen Sie es auf true, um dieses Zertifikat als Vertrauensanker festzulegen.


Auslöser
ArgumentError — Wenn das Zertifikat nicht hinzugefügt werden kann.

connect

()Methode 
override public function connect(host:String, port:int):void

Sprachversion: ActionScript 3.0
Laufzeitversionen: Flash Player 11, AIR 2

Verbindet den Socket über SSL oder TLS mit dem angegebenen Host und Port.

Wenn Sie die SecureSocket.connect()-Methode aufrufen, probiert das Socket, den SSL/TLS-Handshake mit dem Server auszuführen. Wenn der Handshake erfolgreich ist, versucht das Socket, das Serverzertifikat zu überprüfen. Wenn das Zertifikat gültig und vertrauenswürdig ist, wird die sichere Socketverbindung eingerichtet und das Socket setzt ein connect-Ereignis ab. Wenn der Handshake fehlschlägt oder das Zertifikat nicht validiert werden kann, setzt der Socket ein IOError-Ereignis ab. Sie können das Ergebnis der Zertifikatvalidierung überprüfen, indem Sie die serverCertificateStatus-Eigenschaft lesen, nachdem das IOError-Ereignis abgesetzt wurde. (Wenn ein connect-Ereignis abgesetzt wird, ist der Zertifikatstatus immer trusted.)

Wenn bereits eine Socketverbindung besteht, wird die vorhandene Verbindung zunächst geschlossen.

Parameter

host:String — Der Name oder die IP-Adresse des Hosts, zu dem eine Verbindung hergestellt werden soll.
 
port:int — Die Portnummer für die herzustellende Verbindung.


Ereignisse
connect:Event — Wird ausgelöst, wenn eine Netzwerkverbindung hergestellt wurde.
 
ioError:IOErrorEvent — Wird ausgelöst, wenn ein Host angegeben wurde und ein I/O-Fehler auftritt, durch den die Verbindung fehlschlägt. Dies schließt SSL/TLS-Handshake-Fehler sowie das Fehlschlagen der Validierung des Hostserverzertifikats ein.
 
securityError:SecurityErrorEvent — Wird ausgelöst, wenn beim Aufrufen von Socket.connect() versucht wird, eine Verbindung zu einem Server herzustellen, der keine Socket-Richtliniendatei bereitstellt, bzw. zu einem Server, dessen Richtliniendatei dem aufrufenden Host keinen Zugriff auf den angegebenen Port gewährt. Weitere Informationen zu Richtliniendateien finden Sie unter „Websitesteuerungen (Richtliniendateien)“ im ActionScript 3.0 Entwicklerhandbuch und im Flash Player Developer Center im Artikel: Sicherheit.

Auslöser
IOError — Wenn Sie keinen Host angeben und die Verbindung fehlschlägt.
 
SecurityError — Wenn Sie einen Socketport unter 0 oder über 65535 angeben.

securityError

 Ereignis

Wird abgesetzt, wenn ein Aufruf von SecureSocket.connect() aufgrund einer Sicherheitsbeschränkung fehlschlägt.

Ein Sicherheitsfehler-Ereignis wird abgesetzt, wenn Code, der in Flash Player oder in AIR außerhalb der Sicherheitssandbox der Anwendung ausgeführt wird, versucht, eine Verbindung zu einem Server in einer anderen Domäne oder zu einem Port unter 1024 herzustellen und der Server keine Richtliniendatei bereitstellt, die diese Verbindung zulässt. Beachten Sie, dass Code, der innerhalb der AIR-Sicherheitssandbox ausgeführt wird, eine Verbindung zu einem Server in jeder beliebigen Domäne und zu Ports unter 1024 herstellen kann, ohne dass eine Richtliniendatei erforderlich ist.

SecureSocketExample.as

Das folgende Beispiel zeigt das Erstellen und Verbinden eines SecureSocket-Objekts. 
package
{
    import flash.display.Sprite;
    import flash.errors.IOError;
    import flash.events.Event;
    import flash.events.IOErrorEvent;
    import flash.net.SecureSocket;
    
    public class SecureSocketExample extends Sprite
    {
        private var secureSocket:SecureSocket = new SecureSocket();
        
        public function SecureSocketExample()
        {
            secureSocket.addEventListener( Event.CONNECT, onConnect )
            secureSocket.addEventListener( IOErrorEvent.IO_ERROR, onError );
            
            try
            {
                secureSocket.connect( "208.77.188.166", 443 );
            }
            catch ( error:Error )
            {
                trace ( error.toString() );
            }
        }
        
        private function onConnect( event:Event ):void
        {
            trace("Connected.");
        }
        
        private function onError( error:IOErrorEvent ):void
        {
            trace( error.text + ", " + secureSocket.serverCertificateStatus );
        }
    }
}
