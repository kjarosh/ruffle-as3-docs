Paketflash.html
Klassepublic class HTMLWindowCreateOptions
VererbungHTMLWindowCreateOptions Object

Laufzeitversionen: AIR 1.0

Diese Klasse definiert die Optionen, die festgelegt werden können, wenn JavaScript, das in einem HTMLLoader-Objekt ausgeführt wird, versucht, durch Aufrufen der window.open()-Methode ein neues HTML-Fenster zu erstellen.

Diese Klasse definiert die Eigenschaften und Methoden für Optionen im features-Parameter, der an die window.open()-Methode in JavaScript übergeben wird.

JavaScript in einem HTML-Dokument (in einem HTMLLoader-Objekt) kann beispielsweise den folgenden Aufruf an window.open() enthalten, in dem der features-Parameter (der dritte Parameter) eine Reihe von Optionen auflistet:

window.open("http://www.adobe.com", "AdobeWindow", "scrollbars=1,menubar=1,location=0,status=0")

Die HTMLWindowCreateOptions-Klasse wird beim Überschreiben der createWindow()-Methode einer Unterklasse der HTMLHost-Klasse verwendet. Das HTMLLoader-Objekt übergibt ein HTMLWindowCreateOptions-Objekt als windowCreateOptions-Parameter der createWindow()-Methode des HTMLHost-Objekts.

Eigenschaftendetails
    

fullscreen

Eigenschaft
public var fullscreen:Boolean = false

Laufzeitversionen: AIR 1.0

Gibt an, ob das Fenster den ganzen Bildschirm ausfüllen soll. Diese Eigenschaft ist auf true gesetzt, wenn der features-String des JavaScript-Aufrufs an die window.open()-Methode "fullscreen", "fullscreen=1" oder "fullscreen=y" enthält.

    

height

Eigenschaft 
public var height:Number = NaN

Laufzeitversionen: AIR 1.0

Legt die gewünschte anfängliche Höhe des neuen Fensters fest. Dies ist auf den height-Wert im features-String des JavaScript-Aufrufs der window.open()-Methode eingestellt. Wenn der Wert NaN ist (Standardeinstellung, wenn kein height-Wert im features-String angegeben ist), wird eine Standardhöhe für das Fenster verwendet.

Inhalt des ActionScript 3.0-Referenzhandbuchs wird in englischer Sprache angezeigt

Nicht alle Teile des ActionScript 3.0-Referenzhandbuchs wurden in alle Sprachen übersetzt. Wenn der Text zu einem Sprachelement nicht übersetzt wurde, wird er auf Englisch angezeigt. Zum Beispiel wurden die Informationen zur ga.controls.HelpBox-Klasse nicht in andere Sprachen übersetzt. In der deutschen Version des Referenzhandbuchs erscheint der Abschnitt zur ga.controls.HelpBox-Klasse deshalb auf Englisch.
