Pacoteflashx.textLayout.edit
Classepublic class TextClipboard
HerançaTextClipboard Object

Versão da linguagem: ActionScript 3.0
Versões de runtime: Flash Player 10, AIR 1.5

A classe TextClipboard copia e cola objetos TextScrap de e para a área de transferência do sistema.

Quando você copia um TextScrap para o TextClipboard, as informações são copiadas na área de transferência do sistema em dois formatos de área de transferência. Um deles é uma string XML que expressa o objeto TextScrap copiado, na sintaxe Text Layout Markup. Este objeto de área de transferência usa o nome de formato: "TEXT_LAYOUT_MARKUP". O segundo formato é uma string de texto sem formatação, que usa o nome padrão Clipboard.TEXT_FORMAT.

Os métodos da classe TextClipboard são funções estáticas, que dispensam a criação de uma instância do TextClipboard.

Detalhes do método

getContents

()método
public static function getContents():flashx.textLayout.edit:TextScrap

Versão da linguagem: ActionScript 3.0
Versões de runtime: Flash Player 10, AIR 1.5

Lê qualquer texto na área de transferência do sistema como um objeto TextScrap.

Se o formato "TEXT_LAYOUT_MARKUP" estiver disponível, este método converterá a string formatada em um TextScrap e a retornará. Caso contrário, se Clipboard.TEXT_Format estiver disponível, este método converterá a string sem formatação em um TextScrap. Se nenhum formato de área de transferência estiver disponível, este método gerará o valor null.

O Flash Player exige que o método getContents() seja chamado em um gerenciador de eventos de colagem. No AIR, essa restrição só se aplica ao conteúdo fora da área de segurança do aplicativo.

flashx.textLayout.edit:TextScrap

setContents

()método 
public static function setContents(textScrap:flashx.textLayout.edit:TextScrap):void

Versão da linguagem: ActionScript 3.0
Versões de runtime: Flash Player 10, AIR 1.5

Coloca um TextScrap na área de transferência do sistema.

O TextScrap é colocado na área de transferência do sistema como uma representação em Text Layout Markup e em texto sem formatação.

O Flash Player requer um evento de usuário (por exemplo, pressionamento de tecla ou clique de mouse) antes de chamar setContents(). No AIR, essa restrição só se aplica ao conteúdo fora da área de segurança do aplicativo.

Parâmetros

textScrap:flashx.textLayout.edit:TextScrap — O TextScrap a ser colado na área de transferência.
