Wersja języka: ActionScript 3.0
Wersje środowiska wykonawczego: AIR 1.0, Flash Player 9, Flash Lite 4

Klasa Error zawiera informacje o błędzie, który wystąpił w skrypcie. Podczas uruchomienia skompilowanego kodu w wersji środowiska wykonawczego Flash z debuggerem w tworzonych aplikacjach ActionScript 3.0, w oknie dialogowym, wyświetlane są wyjątki typu Error lub podklasa, które umożliwiają usuwanie problemów związanych z kodem. Za pomocą funkcji konstruktora Error można utworzyć obiekt Error. Zwykle nowy obiekt Error jest tworzony w bloku kodu try, który zostaje przechwycony przez blok kodu catch.

Można również utworzyć podklasę klasy Error i wygenerować instancję tej podklasy.

     TypeError: Error #1009: Cannot access a property or method of a null object reference
         at com.xyz::OrderEntry/retrieveData()[/src/com/xyz/OrderEntry.as:995]
         at com.xyz::OrderEntry/init()[/src/com/xyz/OrderEntry.as:200]
         at com.xyz::OrderEntry()[/src/com/xyz/OrderEntry.as:148]

Powyższy kod przedstawia wartość tej metody w przypadku wywołania jej w programie Flash Player z włączonym debugerem lub w programie ADL (AIR Debug Launcher). Jeśli ten kod jest uruchomiony w standardowej wersji programu Flash Player lub środowiska AIR, stos wywołań jest podawany bez ścieżki do pliku i informacji o numerze wiersza — jak na poniższym przykładzie.

     TypeError: Error #1009: Cannot access a property or method of a null object reference
         at com.xyz::OrderEntry/retrieveData()
         at com.xyz::OrderEntry/init()
         at com.xyz::OrderEntry()

W przypadku programu Flash Player 11.4 lub starszego oraz środowiska AIR 3.4 lub starszego stosy wywołań są dostępne tylko wtedy, gdy kod jest uruchomiony w programie Flash Player w wersji z debugerem lub w programie ADL (AIR Debug Launcher). Wywołanie tej metody w wersjach tych środowisk wykonawczych bez debugera powoduje zwrócenie wartości null.

Przykład  ( Sposób korzystania z tego przykładu )

Poniższy przykład ilustruje utworzenie nowego obiektu Error o nazwieerr, a następnie przypisanie ciągu znaków „New Error Message” do obiektu err za pomocą konstruktora Error(). Na końcu właściwość message jest ustawiona na wartość „Another New Error Message”, która nadpisuje wartość „New Error Message”. 

var err:Error = new Error();
trace(err.toString());    // Error

err = new Error("New Error Message");
trace(err.toString());    // Error: New Error Message

err.message = "Another New Error Message";
trace(err.toString());    // Error: Another New Error Message
Przykłady Sposób korzystania z tego przykładu
ErrorExample.as

Poniższy przykład ilustruje użycie klasy ErrorExample w celu wygenerowania błędu niestandardowego. Proces przebiega w następujący sposób:
  1. Lokalna zmienna nullArray typu Array zostaje zadeklarowana bez utworzenia nowego obiektu Array.
  2. Konstruktor próbuje załadować wartość do nieistniejącej tablicy za pomocą metody push() wraz z segmentem kodu obsługującego błąd, który przechwytuje niestandardowy błąd za pomocą klasy CustomError, co powoduje rozszerzenie obiektu Error.
  3. Konstruktor przechwytuje wywołaną klasę CustomError, a następnie za pomocą instrukcji trace() generuje komunikat o błędzie. 
package
{
    import flash.display.Sprite;
    public class ErrorExample extends Sprite 
    {
        private var nullArray:Array;
        public function ErrorExample() 
        {
            try 
            {
                nullArray.push("item");
            }
            catch(e:Error) 
            {
                throw new CustomError("nullArray is null");
            }
        }
    }
}

class CustomError extends Error 
{
    public function CustomError(message:String) 
    {
        super(message);
    }
}
