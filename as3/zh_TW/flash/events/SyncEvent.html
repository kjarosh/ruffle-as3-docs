changeList

屬性
changeList:Array

語言版本: ActionScript 3.0
執行階段版本: AIR 1.0, Flash Player 9, Flash Lite 4

物件陣列，其中每個物件都包含可描述遠端共享物件之變更成員的屬性。 每個物件的屬性都是 code、name 和 oldValue。

當您初次連線至在本機或伺服器上存放的遠端共享物件時，此物件的所有屬性都會設定為空字串。

否則，Flash 便會將 code 設定為 "clear"、"success"、"reject"、"change" 或 "delete"。

  • 值為 "clear" 表示您已成功連線至並非存放在伺服器或用戶端上的遠端共享物件，或者此物件的所有屬性都已刪除。例如，當用戶端和伺服器上物件副本不同步的情況過於嚴重，以致 Flash Player 必須重新將用戶端物件與伺服器物件同步化時的情況。 在後者的情況下，會傳送 SyncEvent.SYNC，並且將 "code" 值設定為 "change"。
  • 值為 "success" 表示用戶端變更了共享物件。
  • 值為 "reject" 表示此用戶端變更物件的嘗試未成功，反倒是該物件遭到其它用戶端變更。
  • 值為 "change" 表示其它用戶端變更了物件，或伺服器已重新將該物件同步化。
  • 值為 "delete" 表示特質已遭刪除。

name 屬性包含已變更之屬性的名稱。

oldValue 屬性包含已變更之屬性的原先值。 除非 code 的值為 "reject" 或 "change"，否則此參數都會是 null。

SyncEvent

()建構函式
public function SyncEvent(type:String, bubbles:Boolean = false, cancelable:Boolean = false, changeList:Array = null)

語言版本: ActionScript 3.0
執行階段版本: AIR 1.0, Flash Player 9, Flash Lite 4

建立包含 sync 事件相關資訊的 Event 物件。 Event 物件會當做參數傳遞至事件偵聽程式。

參數
type:String — 事件類型。事件偵聽程式可以透過繼承的 type 屬性來存取此資訊。sync 事件只有一種類型：SyncEvent.SYNC。
 
bubbles:Boolean (default = false) — 判斷 Event 物件是否參與事件流程的反昇階段。事件偵聽程式可以透過繼承的 bubbles 屬性來存取此資訊。
 
cancelable:Boolean (default = false) — 判斷是否可以取消 Event 物件。事件偵聽程式可以透過繼承的 cancelable 屬性來存取此資訊。
 
changeList:Array (default = null) — 物件陣列，可說明與遠端 SharedObject 的同步處理。 事件偵聽程式可以透過 changeList 屬性來存取此物件。
