在視訊播放程式調整大小和/或配置時傳送的事件。 以下是兩種配置案例：
  • 如果配置視訊播放程式的方式是使用 autoLayout 事件、呼叫 setScale() 或 setSize() 方法，或是變更 width、height、scaleX、scaleY、registrationWidth 和 registrationHeight 屬性。
  • 如果有兩個不同大小或位置的視訊播放程式，而且 visibleVideoPlayerIndex 屬性從其中一個視訊播放程式切換至另一個播放程式。

LayoutEvent 物件只有一種類型：LayoutEvent.LAYOUT。

LayoutEvent

()建構函式
public function LayoutEvent(type:String, bubbles:Boolean = false, cancelable:Boolean = false, oldBounds:Rectangle = null, oldRegistrationBounds:Rectangle = null)

語言版本: ActionScript 3.0
產品版本: Flash CS3
執行階段版本: Flash Player 9.0.28.0, AIR 1.0

會建立包含 layout 事件相關資訊的 Event 物件。 Event 物件會當做參數傳遞至事件偵聽程式。

參數
type:String — 事件類型。事件偵聽程式可以透過繼承的 type 屬性來存取此資訊。自動配置事件只有一種類型：LayoutEvent.LAYOUT。
 
bubbles:Boolean (default = false) — 判斷 Event 物件是否參與事件流程的反昇階段。事件偵聽程式可以透過繼承的 bubbles 屬性來存取此資訊。
 
cancelable:Boolean (default = false) — 判斷是否可以取消 Event 物件。事件偵聽程式可以透過繼承的 cancelable 屬性來存取此資訊。
 
oldBounds:Rectangle (default = null) — 會指出 layout 事件發生之前，目標的 x、y、width 和 height 屬性值。 事件偵聽程式可以透過 oldBounds 屬性來存取此資訊。
 
oldRegistrationBounds:Rectangle (default = null) — 會指出 layout 事件發生之前，目標的 registrationX、registrationY、registrationWidth 和 registrationHeight 屬性值。 事件偵聽程式可以透過 oldRegistrationBounds 屬性來存取此資訊。
