編譯器警告訊息會識別有效且成功編譯，但可能不是作者想要的程式碼。若要偵測這些可能的問題，請在警告模式中編譯 ActionScript 專案。

某些警告 (例如，「遺失類型宣告」) 是您可以選擇是否要執行的編碼樣式選擇。其它警告 (例如，「無法指定為 null」) 則會指出有效、但可能無法如使用者預期般運作的陳述式。第三種警告涵蓋了將 ActionScript 2.0 程式碼匯入 ActionScript 3.0 時可能遇到的問題。


 程式碼訊息說明
 1008遺失類型宣告。 
 1009%s '%s' 沒有類型宣告。 不宣告資料類型是一種屬於個人偏好的程式撰寫風格。函數傳回沒有類型宣告的類型、參數或變數。然而，使用類型宣告可讓編譯器撰寫更有效率的程式碼，同時還能在編譯階段偵測更多錯誤。如果您希望在未使用類型宣告時出現此提示提醒您，請啟用此警告。  
 1012不合邏輯的未定義項目比較。只有不具類型的變數 (或類型 * 的變數) 可不加以定義。 
 1013必須定義類型 %s 的變數。未定義的值會是比較之前強制轉型成 %s 的類型。 只有類型 * 的變數可不加以定義。在少數例外情況下，未初始化的變數具有預設值 null，而非 undefined。這些例外包括：Boolean 變數，其預設值為 false。Number 變數，其預設值為 NaN；int 或 uint 變數，其預設值為 0。  
 1030新運算式中使用的函數會傳回值。結果將會是函數傳回的值，而不是該函數的新實體。 
 1031移轉問題：新 %s 的結果將會是 %s 的傳回值，而不是該函數的新實體。 這是程式碼移轉警告。偵測到的程式碼在 ActionScript 3.0 的行為與在 ActionScript 2.0 中不同，如下列範例所示： 
function f(){
   this.b = 22;
   this.a = new Array(2);Â  
   this.a[0] = 33;
   this.a[1] = 44;
   return a; 
   } 
   // returns a new instance of f in ActionScript 2.0 and a new 2 element array in ActionScript 3.0
   var d = new f();Â  // Warning here
   trace(d.a);       // undefined in ActionScript 3.0, [33,44] in ActionScript 2.0.
 
 1034在 ActionScript 3.0 中，不含引數的 Boolean() 會傳回 false。在 ActionScript 2.0 中，Boolean() 會傳回 undefined。 
 1035使用不含引數的 Boolean()。 這是程式碼移轉警告。Boolean() 函數在 ActionScript 3.0 中會傳回 false，但在 ActionScript 2.0 中會傳回 undefined。  
 1038在 ActionScript 3.0 中，空格會被忽略，且 '' 會傳回 0。而在 ActionScript 2.0 中，當參數為 '' 或包含空格時，Number() 會傳回 NaN。 
 1039移轉問題：使用空白字串引數呼叫 Number('') 函數時，在 ActionScript 3.0 中會傳回 0，而在 ActionScript 2.0 中則會傳回 NaN。 這是程式碼移轉警告。使用 String 引數呼叫 Number() 方法時會略過字串中的所有空格，若沒有偵測到數字，則會傳回預設值 0。在 ActionScript 2.0 中，如果字串中有任何空格都會造成 NaN 結果。  
 1044Array.toString() 格式已變更。 
 1045移轉問題：Array.toString() 處理 null 和未定義元素的方式已變更。 這是程式碼移轉警告。在 ActionScript 2.0 中，null 陣列元素會轉換為 null，而 undefined 元素會轉換為 undefined。在 ActionScript 3.0 中，null 和 undefined 元素都會轉換為空白字串 ''。如果您具有可從 Array 剖析 toString() 輸出的程式碼，則可能需要根據此差異調整您的程式碼。

 1072類別已密封。無法以動態方式加入成員。 
 1073移轉問題：%s 不是動態類別。無法以動態方式讓成員加入實體。 這是程式碼移轉警告。在 ActionScript 2.0 中，許多如 Number 的類別都是動態的，也就是說新的屬性可以在執行階段加入那些類別的實體。之所以出現這個警告，是因為程式碼嘗試將屬性新增至非動態類別的實體中。  
 1082this 關鍵字的範圍變更。從類別實體擷取而來的類別方法一定會將 this 解析回該實體。在 ActionScript 2.0 中，會根據叫用方法的位置來動態查詢 this。 
 1083移轉問題：在 ActionScript 3.0 中，方法 %s 會因 this 關鍵字的範圍變更而有不同的行為。如需詳細資訊，請參閱警告 1083。 這是程式碼移轉警告。這個警告會在物件的方法被當作值 (通常當作回呼函數) 使用時產生。在 ActionScript 2.0 中，函數是在被呼叫的內容中執行。在 ActionScript 3.0 中，函數一定會在被定義的內容中執行。因此，變數和方法名稱會解析成屬於回呼一部分的類別，而不是相對於被呼叫的內容，如下列範例所示： 
class a 
{ 
   var x; 
   function a() { x = 1; } 
   function b() { trace(x); } 
}

var A:a = new a();
var f:Function = a.b; // warning triggered here
var x = 22;
f(); // prints 1 in ActionScript 3.0, 22 in ActionScript 2.0
 
 1084遺失命名空間宣告 (例如變數未定義為 public、private 等)。 
 1085%s 範圍將為預設的名稱空間：%s 內部。在此套件外將不會看到它。 不宣告名稱空間是一種屬於個人偏好的程式撰寫風格。如果您希望在忘記對定義宣告名稱空間或存取指定字時出現提示提醒您，請啟用此警告。若沒有，位於此檔案外的程式碼便看不到定義。如果要讓位於此檔案外的程式碼看得到定義，請使用存取指定字 public 或名稱空間宣告來宣告它。如果要讓定義屬於這個檔案，並避免出現這個警告，請將定義宣告為 private。  
 1086ActionScript 3.0 會以隨機順序在 "for x in target" 陳述式內重複執行物件的屬性。 
 1087移轉問題：ActionScript 3.0 會以隨機順序在 "for x in target" 陳述式內重複執行物件的屬性。 這是程式碼移轉警告。在 ActionScript 2.0 中，處理物件屬性的順序一直都是相同的。在 ActionScript 3.0 中，則是以隨機順序處理，且會依照使用的電腦而變更。如果發生不可預期的順序行為，請檢查此迴圈，以判斷此行為中的變更是否會影響您的程式碼。

 1096不合邏輯的 null 項目比較。 
 1097不合邏輯的 null 項目比較。類型 %s 的變數不可以是 null。 Boolean、int、uint 和 Number 的實體不可以是 null。比較運算子類型在與 Boolean 比較前會先將 null 轉換為 false，或在與 Number、int 或 uint 資料類型比較前先轉換為 0。  
 1098不合邏輯的 NaN 項目比較。任何牽涉到 NaN 的比較運算都會評估為 false，因為 NaN != NaN。 
 1099不合邏輯的 NaN 項目比較。這個陳述式永遠會評估為 false。 NaN 具有獨特的數學屬性，任何與其關聯的比較都會評估為 false。請改用全域 isNaN() 函數來偵測 NaN 值，如下列範例所示： 
trace(NaN == NaN); // false!
trace(NaN != NaN); // false again!
trace(isNaN(NaN)); // true
 
 1100在條件內指定。 
 1101在條件內指定。您是指以 == 取代 =？ = 指定陳述式的結果即為 = 陳述式右邊的值。您可以將指定陳述式當作條件測試，但我們不建議這個作法。因為這種情況通常是打錯字的結果，原本應該是 == 相等測試，如下列範例所示： 
var x:Boolean = false;
var y:Boolean = true;
// it is hard to determine if the line below intentionally sets x's value to y's or if its a typo
if (x = y) { trace("x is assigned y's value of true, making the conditional test evaluate as true."); }
