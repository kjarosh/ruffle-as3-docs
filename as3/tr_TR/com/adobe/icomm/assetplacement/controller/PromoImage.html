A PromoImage is an Image control that gets its source value by querying the PackageDefinition object (which is just a wrapper for the package definition file - package.xml) during initialization.

The component's id attribute is the key that is used to lookup related assets, in the form of Resource objects, in the PackageDefinition. When the id is set, the object will automatically query the PackageDefinition for resources related to that id.

Note that the id lookup in the PackageDefinition is the only way to change the source value of this object. Attempts to set the source property manually will be ignored.

PromoImage is set to autoLoad its content during initialization. If autoLoad is set to false then an explicit call to the load() method is necessary to force then content to load. Note that you cannot change the source of the content by changing the autoLoad setting and manually calling load(); this merely delays the loading of the content - it will still load the content pointed to by the source attribute of the package definition file's <resource/> entry for this object.

PromoImage also implements some helper functionality for setting a targetURL that will be automatically invoked when the PromoImage control is clicked. By default, this behavior is on and the targetURL is loaded from the PackageDefinition. This functionality can be disabled by setting autoInvokeTargetURL to false.

Identifies how an image is to be scaled.

The possible values are:

  • SCALING_METHOD_ACTUAL: (Default) The image shall be rendered using the dimensions stored in the image content.
  • SCALING_METHOD_FIT: The processing application must scale the image proportionally to the maximum possible size such that it fits within the nominal content region of the marketing cell.
  • SCALING_METHOD_SHOWALL: Specifies that the entire image to be visible in the specified area without distortion while maintaining the original aspect ratio of the image
