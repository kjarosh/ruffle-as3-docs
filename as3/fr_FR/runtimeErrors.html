 1061Il est impossible de convertir la valeur %1 en %2 sans perte de précision. Cette erreur se produit lorsque vous tentez d’affecter un nombre décimal à une propriété dont le type de données est int.

Cette erreur se produit également pour des affectations hors limites, comme dans l’exemple suivant :

var m0:int = 2147483648; // int.MAX_VALUE == 2147483647

Vous pouvez également rencontrer cette erreur lorsque vous utilisez l’opérateur de décalage gauche au niveau du bit (1). Considérons par exemple le code qui suit :

var m0:uint = 0xFF;
var m1:uint = m0<<24;

Le résultat de l’opérateur de décalage vers la gauche (<<) est interprété en tant que nombre 32 bits de type complément à deux avec signe. Dans cet exemple, le résultat est une valeur négative qui déclenche l’erreur lorsqu’elle est affectée à la propriété de type uint. Vous pouvez utiliser la solution suivante :

var m0:uint = 0xFF;
var m1:uint = uint(m0<<24);

Tout le contenu du Guide de référence ActionScript 3.0 n’a pas été traduit dans toutes les langues. Si un élément de langage n’est pas traduit, il s’affiche en anglais.
