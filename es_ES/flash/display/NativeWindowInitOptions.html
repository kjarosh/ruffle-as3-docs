Clasepublic class NativeWindowInitOptions
HerenciaNativeWindowInitOptions Object

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: AIR 1.0

La clase NativeWindowInitOptions define las opciones de inicialización utilizadas para construir una nueva instancia de NativeWindow.

Las propiedades definidas en las opciones de inicialización no se pueden modificar una vez creada la ventana.

Nota: en la ventana inicial de la aplicación creada automáticamente por AIR, todas las propiedades (excepto type) se establecen en el descriptor de la aplicación. La ventana inicial siempre es de tipo NativeWindowType.NORMAL.

maximizable

propiedad
maximizable:Boolean

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: AIR 1.0

Indica si el usuario puede maximizar la ventana.

En ventanas con fondo cromático del sistema, la configuración afecta a la apariencia del botón Maximizar de la ventana. También afecta a otras partes de la interfaz de usuario administrada por el sistema, como los menús de las ventanas de Microsoft Windows.

Si se establece como false, el usuario no puede maximizar la ventana. Llamar directamente al método maximize() de NativeWindow maximizará la ventana.

Notas sobre el comportamiento del SO:

  • En sistemas operativos como Mac OS X, donde al maximizar una ventana se puede cambiar su tamaño, tanto maximizable como resizable se deben establecer en false para impedir que la ventana se pueda ampliar o cambiar de tamaño.
  • Algunos gestores de ventanas de Linux permiten que el usuario maximice las ventanas incluso cuando la propiedad maximizable se establece en false.

El valor predeterminado es true.

Cuando una ventana tiene propietario, dicha ventana siempre se muestra delante de su propietario, está minimizada, se oculta junto con su propietario, y se cierra cuando se cierra su propietario.

resizable

propiedad 
resizable:Boolean

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: AIR 1.0

Indica si el usuario puede cambiar el tamaño de la ventana.

Si se establece en false, el usaurio no puede cambiar el tamaño de la ventana con el fondo cromático del sistema. Llamar al método startResize() de NativeWindow como respuesta a un evento de ratón permitirá que el usuario pueda cambiar el tamaño de la ventana. Establecer los límites de la ventana directamente también cambia el tamaño de la ventana.

Notas sobre el comportamiento del SO:

  • En sistemas operativos como Mac OS X, donde maximizar ventanas es una operación de cambio de tamaño, tanto maximizable como resizable se deben establecer en false para impedir que la ventana se pueda ampliar o cambiar de tamaño.
  • Algunos gestores de ventanas de Linux permiten que el usuario cambie el tamaño de las ventanas incluso cuando la propiedad resizable se establece en false.

El valor predeterminado es true.

systemChrome

propiedad 
systemChrome:String

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: AIR 1.0

Especifica si se suministra el fondo cromático del sistema para la ventana.

Entendemos por control de aspecto los controles de la ventana que permiten al usuario definir las propiedades de escritorio de una ventana. Los controles de aspecto del sistema utilizan los controles estándar del entorno del escritorio en el que se ejecuta la aplicación de AIR y se ajustan a la apariencia del sistema operativo nativo.

Para utilizar los controles de aspecto del sistema proporcionados por un marco de trabajo (por ejemplo, Flex), o para crear controles de aspecto propios de la ventana, establezca systemChrome como NativeWindowSystemChrome.NONE.

Las constantes de los valores válidos de esta propiedad se definen en la clase NativeWindowSystemChrome:

  • NativeWindowSystemChrome.NONE
  • NativeWindowSystemChrome.STANDARD

Si no especifica, el valor predeterminado de systemChrome es NativeWindowSystemChrome.STANDARD.

No se puede establecer la propiedad transparent como true en ventanas con fondo cromático del sistema.

El valor predeterminado es NativeWindowSystemChrome.STANDARD.

transparent

propiedad 
transparent:Boolean

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: AIR 1.0

Indica si la ventana admite transparencia y mezcla alfa en el escritorio.

Si es true, la ventana se muestra superpuesta al escritorio. Las áreas de la ventana no cubiertas por un objeto de visualización, o cubiertas por objetos de visualización con configuración de alfa cercana a cero, se comportan como invisibles y no detectan eventos de ratón (que recibiría el objeto de escritorio situado debajo de la ventana). El valor alfa en el que un objeto deja de detectar eventos de ratón oscila entre 0,06 y 0,01, en función del sistema operativo.

No se puede establecer la propiedad transparent como true en ventanas con fondo cromático del sistema.

Nota: no todos los gestores de ventanas de Linux admiten transparencia. En dichos sistemas, las áreas transparentes de una ventana se visualizan en negro.

El valor predeterminado es false.

type

propiedad 
type:String

Versión del lenguaje: ActionScript 3.0
Versiones de motor de ejecución: AIR 1.0

Especifica el tipo de ventana que se va a crear.

Las constantes de los valores válidos de esta propiedad se definen en la clase NativeWindowType:

  • NativeWindowType.NORMAL: una ventana normal. Las ventanas normales utilizan el fondo cromático completo y se muestran en la barra de tareas de de Windows o de Linux.
  • NativeWindowType.UTILITY: paleta de herramientas. Las ventanas de utilidades utilizan una versión más ligera de los controles de aspecto del sistema y no se muestran en la barra de tareas de Windows.
  • NativeWindowType.LIGHTWEIGHT: las ventanas ligeras no pueden tener fondo cromático del sistema y no aparecen en la barra de tareas de Windows o de Linux. Además, las ventanas ligeras no disponen de menú Sistema (Alt-Barra espaciadora) en Windows. El uso de ventanas ligeras está recomendado para mensajes emergentes de notificación o controles como cuadros emergentes áreas de visualización temporal. Si utiliza ventanas ligeras, systemChrome debe establecerse en NativeWindowSystemChrome.NONE.

Si no se especifica ningún valor, el valor predeterminado de type es NativeWindowType.NORMAL.

El valor predeterminado es NativeWindowType.NORMAL.
