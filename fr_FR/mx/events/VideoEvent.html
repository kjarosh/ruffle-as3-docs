Packagemx.events
Classepublic class VideoEvent
HéritageVideoEvent Event Object

Version du langage: ActionScript 3.0
Version du produit: Flex 3
Versions du moteur d’exécution: Flash Player 9, AIR 1.1

La classe VideoEvent représente l’objet d’événement transmis à l’écouteur d’événement pour les événements distribués par le contrôle VideoDisplay et définit les valeurs de la propriété VideoDisplay.state.

Constructeur.

Paramètres
type:String — Type d’événement, indiquant quelle action a déclenché l’événement.
 
bubbles:Boolean (default = false) — Détermine si l’événement peut se propager vers le haut de la hiérarchie de la liste d’affichage.
 
cancelable:Boolean (default = false) — Indique si le comportement associé à l’événement peut être évité.

playheadTime:Number (default = NaN) — Position de la tête de lecture au moment où l’événement se produit.

CONNECTION_ERROR

Constante 
public static const CONNECTION_ERROR:String = "connectionError"

Version du langage: ActionScript 3.0
Version du produit: Flex 3
Versions du moteur d’exécution: Flash Player 9, AIR 1.1

Valeur de la propriété VideoDisplay.state au moment où le contrôle VideoDisplay n’a pas réussi à charger le flux vidéo. Cet état peut se produire lorsqu’aucune connexion à un serveur n’est établie, lorsque le flux vidéo est introuvable ou encore pour d’autres raisons.

Il s’agit d’un état non réactif. Si le contrôle est non réactif, les appels des méthodes play(), load(), stop() et pause() sont mis en file d’attente, puis exécutés lorsque le contrôle passe à l’état réactif.

Certaines parties du Guide de référence ActionScript 3.0 sont en anglais

Tout le contenu du Guide de référence ActionScript 3.0 n’a pas été traduit dans toutes les langues. Si un élément de langage n’est pas traduit, il s’affiche en anglais.
