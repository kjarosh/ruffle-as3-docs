A partir de Flex 4.0, Adobe vous recommande d’utiliser la classe spark.components.TextInput comme alternative à cette classe.

Le contrôle TextInput est un champ de texte d’une seule ligne qui est entièrement modifiable. Tout le texte de ce contrôle doit utiliser les mêmes styles, sauf s’il s’agit d’un texte HTML. Le contrôle TextInput prend en charge les fonctionnalités de rendu HTML de Flash Player et AIR.

Les contrôles TextInput n’incluent pas d’étiquette, mais vous pouvez en ajouter une à l’aide d’un contrôle Label ou en incorporant le contrôle TextInput dans un contrôle FormItem d’un conteneur Form. Lorsqu’il est utilisé dans un contrôle FormItem, un contrôle TextInput indique si une valeur est requise. Les contrôles TextInput ont différents états, notamment ils peuvent être remplis, sélectionnés, désactivés ou contenir une erreur. Ils prennent en charge la mise en forme, la validation et les équivalents clavier. Ils distribuent également les événements change et enter.

Si vous désactivez un contrôle TextInput, il affiche son contenu dans la couleur indiquée par le style disabledColor. Pour désactiver l’édition du texte, définissez la propriété editable sur false. Pour masquer le texte de saisie en affichant des astérisques à la place des caractères saisis, définissez la propriété displayAsPassword sur true.

Le contrôle TextInput est utilisé comme sous-composant dans plusieurs autres contrôles, comme les contrôles RichTextEditor, NumericStepper et ComboBox. Par conséquent, si vous affectez des propriétés de style à un contrôle TextInput à l’aide d’un sélecteur de type CSS, Flex applique ces styles au contrôle TextInput lorsqu’il s’affiche dans les autres contrôles, sauf si vous les avez explicitement remplacés.

htmlText

propriété 
htmlText:String

Version du langage: ActionScript 3.0
Version du produit: Flex 3
Versions du moteur d’exécution: Flash Player 9, AIR 1.1

Spécifie le texte affiché par le contrôle TextInput, y compris la balise HTML définissant les styles du texte. Lorsque vous spécifiez du texte HTML dans cette propriété, vous pouvez utiliser le sous-jeu de balises HTML pris en charge par le contrôle Flash TextField.

Lorsque vous définissez cette propriété, le marqueur HTML est appliqué après l’application des styles CSS pour l’instance TextInput. Lorsque vous obtenez cette propriété, le marqueur HTML inclut les styles CSS.

Par exemple, si vous le définissez en tant que chaîne (par exemple « Ceci est un exemple de marqueur <b>bold</b> »), le texte « Ceci est un exemple de marqueur bold » s’affiche dans le contrôle TextInput avec les styles CSS qui s’appliquent normalement. De plus, le terme « bold » (gras) s’affiche en gras à cause du marqueur <b>.

Les marqueurs HTML utilisent des caractères comme < et >, qui ont une signification particulière en langage XML (et par conséquent MXML). Ainsi, le code suivant n’est pas conforme pour la compilation :

      <mx:TextInput htmlText="This is an example of <b>bold</b> markup"/>

Trois méthodes permettent de contourner ce problème.

  • Définissez la propriété htmlText dans une méthode ActionScript appelée comme gestionnaire initialize :

          <mx:TextInput id="myTextInput" initialize="myTextInput_initialize()"/>

    dont la méthode TextInput_initialize se trouve dans la section CDATA d’un script:

          <fx:Script>
      <![CDATA[
      private function myTextInput_initialize():void {
          myTextInput.htmlText = "This is an example of <b>bold</b> markup";
      }
      ]]>
      </fx:Script>

    Ceci est l’approche la plus simple, car le marqueur HTML reste facilement lisible. Notez que vous devez attribuer un id au contrôle TextInput afin de pouvoir vous y référer dans le gestionnaire initialize.

  • Spécifiez la propriété htmlText en utilisant une balise enfant avec une section CDATA. Une section CDATA en langage XML contient des données de caractères pour lesquelles les caractères comme < et > n’ont pas de signification particulière.

          <mx:TextInput>
          <mx:htmlText><![CDATA[This is an example of <b>bold</b> markup]]></mx:htmlText>
      <mx:TextInput/>

    Vous devez écrire la propriété htmlText en tant que balise enfant au lieu d’un attribut sur la balise <mx:TextInput>, car le langage XML n’autorise pas la section CDATA comme valeur d’attribut. Notez que le marqueur est lisible, mais la section CDATA rend cette approche plus compliquée.

  • Utilisez un attribut hmtlText dont toutes les instances de caractères de marqueur HTML < et > dans la valeur d’attribut sont écrites, au lieu d’utiliser les « entités » XML < et > :

          <mx:TextInput htmlText="This is an example of <b>bold</b> markup"/>
    Adobe ne recommande pas cette approche, car le marqueur HTML devient presque impossible à lire.

Si la propriété condenseWhite a la valeur true lorsque vous définissez la propriété htmlText, plusieurs caractères d’espace sont condensés, comme dans les navigateurs HTML. Par exemple, trois espaces consécutifs sont affichés sous la forme d’un espace unique. La valeur par défaut de condenseWhite est false. Vous devez donc définir condenseWhite sur true pour réduire l’espace.

Si vous relisez la propriété htmlText peu de temps après l’avoir définie, vous obtenez la même chaîne que celle que vous avez définie. Toutefois, après l’exécution de l’objet LayoutManager, la valeur change pour inclure un marqueur supplémentaire comprenant les styles CSS.

La définition de la propriété htmlText affecte la propriété text de plusieurs manières. Si vous lisez la propriété text peu de temps après avoir défini la propriété htmlText, vous obtenez la valeur null. Cela indique que la propriété text correspondant à la nouvelle propriété htmlText n’a pas encore été déterminée. Toutefois, après l’exécution de l’objet LayoutManager, la propriété text se transforme en chaîne htmlText et tous les marqueurs HTML sont supprimés. En fait, la valeur correspond aux caractères que le contrôle TextInput affiche réellement.

A l’inverse, si vous définissez la propriété text, toute propriété htmlText précédemment définie devient obsolète. Si vous lisez la propriété htmlText peu de temps après avoir défini la propriété text, vous obtenez la valeur null. Cela indique que la propriété htmlText qui correspond à la nouvelle propriété text n’a pas encore été déterminée. Toutefois, après l’exécution du gestionnaire LayoutManager, la valeur de la propriété htmlText devient le nouveau texte et inclut un marqueur supplémentaire pour les styles CSS.

Pour que le gestionnaire LayoutManager s’exécute immédiatement, vous pouvez appeler la méthode validateNow() sur le contrôle TextInput. Par exemple, vous pouvez obtenir des propriétés htmlText, appeler la méthode validateNow() et récupérer immédiatement la propriété text correspondante qui n’a pas le marqueur HTML.

Si vous définissez les deux propriétés text et htmlText dans ActionScript, celle définie en dernier prend effet. Ne définissez pas les deux dans MXML, car MXML ne garantit pas que les propriétés d’une instance puissent être définies dans un ordre particulier.

Définir la propriété text ou htmlText à l’intérieur d’une boucle est une opération rapide, car le contrôle TextField sous-jacent qui effectue réellement le rendu du texte n’est pas mis à jour tant que le gestionnaire LayoutManager s’exécute.

Si vous essayez de définir cette propriété sur null, elle est à la place définie sur la chaîne vide. Si la propriété a temporairement la valeur null, cela indique que la propriété text a été récemment définie et que la propriété htmlText correspondante n’a pas encore été déterminée.

La valeur par défaut est "".

Cette propriété peut être utilisée comme source de la liaison de données. Lorsque cette propriété est modifiée, elle distribue l’événement htmlTextChanged .

Pour afficher un texte mis en forme à l’aide de balises HTML, utilisez plutôt la propriété htmlText. Si vous définissez la propriété htmlText, le code HTML remplace le texte défini à l’aide de cette propriété et la propriété text renvoie une version de texte ordinaire du texte HTML, dépouillée de toutes les balises HTML. Pour plus d’informations, voir la propriété htmlText.

Pour inclure les caractères spéciaux de signe inférieur à (<), de signe supérieur à (>) ou de perluète (&) dans le texte, enveloppez la chaîne de caractères dans la balise CDATA. Vous pouvez également utiliser des entités de caractères HTML pour les caractères spéciaux, par exemple <.

Si vous essayez de définir cette propriété sur null, elle est à la place définie sur la chaîne vide. La propriété text peut avoir temporairement la valeur null, ce qui indique que la propriété htmlText a été récemment définie et que la propriété text n’a pas encore été déterminée.

Cette propriété peut être utilisée comme source de la liaison de données. Lorsque cette propriété est modifiée, elle distribue l’événement textChanged .
