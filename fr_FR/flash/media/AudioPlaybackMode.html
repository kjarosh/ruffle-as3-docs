Packageflash.media
Classepublic final class AudioPlaybackMode
HéritageAudioPlaybackMode Object

Version du langage: ActionScript 3.0
Versions du moteur d’exécution: AIR 3.0

La classe AudioPlaybackMode définit les constantes pour la propriété audioPlaybackMode de la classe SoundMixer.

Problème connu : aucune donnée audio n’est lue dans le mode de lecture de StageVideo, lorsque la mise en sourdine est activée.

Chacune de ces constantes représente un ensemble de comportements audio sur les périphériques mobiles destinés à une utilisation particulière.

Détails de la constante
    

AMBIENT

Constante
public static const AMBIENT:String = "ambient"

Version du langage: ActionScript 3.0
Versions du moteur d’exécution: AIR 3.4

Mode de lecture du son d’ambiance. Utilisez ce mode pour lire les sons qui accompagnent l’application et ne sont pas nécessaires à son bon fonctionnement. Lorsque vous utilisez ce mode, le son est acheminé via le haut-parleur. Lors du raccordement d’un casque, celui-ci remplace le haut-parleur. Le système d’exploitation utilise les valeurs par défaut pour le son d’ambiance lorsqu’il est nécessaire d’établir le comportement et la priorité des sons.

Ce paramètre n’a aucun effet particulier sur le bureau ou le téléviseur.

Sur iOS, l’utilisation de ce mode coupe le son en verrouillant l’écran (ou en envoyant l’application en arrière-plan par d’autres moyens) et en activant l’interrupteur de coupure du son du matériel lorsque le microphone n’est pas en cours d’utilisation. Lors de l’utilisation d’un microphone, ces actions n’ont aucun effet sur le son.

Remarque : notez que le son n’est pas coupé par l’interrupteur de coupure du son lors de la connexion d’un casque.

    

MEDIA

Constante 
public static const MEDIA:String = "media"

Version du langage: ActionScript 3.0
Versions du moteur d’exécution: AIR 3.0

Mode pour la lecture des sons multimédias. Lorsque vous utilisez ce mode sur un périphérique mobile, le son est acheminé via le haut-parleur. Lors du raccordement d’un casque, celui-ci remplace le haut-parleur. Le système d’exploitation utilise les valeurs par défaut pour les sons multimédias lorsqu’il est nécessaire d’établir le comportement et la priorité des sons.

Ce paramètre n’a aucun effet particulier sur le bureau ou le téléviseur.

Il s’agit de la valeur par défaut.

    

VOICE

Constante 
public static const VOICE:String = "voice"

Version du langage: ActionScript 3.0
Versions du moteur d’exécution: AIR 3.0

Mode pour la lecture audio vocale. Lorsque vous utilisez ce mode, le son est par défaut acheminé via l’écouteur du téléphone. Lors du raccordement d’un casque, celui-ci remplace les écouteurs. Si la propriété SoundMixer.useSpeakerphoneForVoice est définie, le son est diffusé via le haut-parleur et non via les écouteurs ou le casque. Le système d’exploitation utilise les valeurs par défaut pour les sons vocaux lorsqu’il est nécessaire d’établir le comportement et la priorité des sons.

Essayez de minimiser l’utilisation du modeAudioPlaybackMode.VOICE et essayez de basculer sur le mode AudioPlaybackMode.MEDIA dès que possible au terme d’un appel vocal afin de permettre la lecture d’autres applications en mode multimédia.

Ce paramètre n’a aucun effet particulier sur le bureau ou le téléviseur.

Certaines parties du Guide de référence ActionScript 3.0 sont en anglais

Tout le contenu du Guide de référence ActionScript 3.0 n’a pas été traduit dans toutes les langues. Si un élément de langage n’est pas traduit, il s’affiche en anglais. Par exemple, la classe ga.controls.HelpBox n’est traduite dans aucune langue. Ainsi, dans la version française du guide de référence, la classe ga.controls.HelpBox apparaît en anglais.
